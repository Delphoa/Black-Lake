# .lake-data

Canonical home for Black Lake DEP deposits.

Use exactly one DEP class per entry:

- `DEP-E-YYYYMMDD-Short Description` for ongoing research.
- `DEP-R-YYYYMMDD-Short Description` for static records.
- `DEP-A-YYYYMMDD-Short Description` for long cold-storage artifacts.

Each DEP entry must include a `README.md`, classification tags, an itemized inventory, item summaries, insights/relevance notes, and a final Attribution Block.

The final commit completing a DEP must start with the DEP data subject title, for example:

```text
Semantically Enhanced PCFG for Password Analysis: add DEP-R-20260708-SEPCFG Paper
```
