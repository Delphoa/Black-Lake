# Prefix Knowledge as a Query-Indexed Attention Surrogate

## A detailed review, technical reconstruction, and independent re-conceptualization of “Context Memorization for Efficient Long Context Generation”

**Source paper:** Yasuyuki Okoshi, Hao Mark Chen, Guanxi Lu, Hongxiang Fan, Masato Motomura, and Daichi Fujiki, arXiv:2605.18226v1, submitted 18 May 2026.[^source]

**Canonical public sources:** [arXiv record](https://arxiv.org/abs/2605.18226), [PDF](https://arxiv.org/pdf/2605.18226), [full-paper HTML](https://arxiv.org/html/2605.18226), and [arXiv-issued DOI](https://doi.org/10.48550/arXiv.2605.18226).

**Review status:** The complete 14-page paper, including appendices, tables, figures, equations, references, and limitations, was reviewed from locally verified PDF and full-paper HTML. External primary records and the paper-linked repository were inspected. No experiment was rerun.

**Evidence model:** **Paper report** identifies what the authors state or measure. **External primary evidence** comes from official paper, benchmark, proceedings, or code records. **Reviewer inference** is analysis derived from those sources. **Hypothesis/proposal** is a new testable extension. These layers remain separate throughout.

## Executive assessment

### The paper in one sentence

**Paper report:** The paper replaces repeated attention over a fixed long prefix with a per-layer dictionary that maps representative query vectors to precomputed prefix-attention outputs and softmax normalization masses, retrieves the nearest entry at inference, and merges it with ordinary self-attention.

### The deeper idea

The exact mathematical fact underneath the paper is modest but powerful: attention over disjoint key–value blocks can be computed independently and combined if each block contributes both its normalized attention output and its unnormalized softmax partition mass. The paper turns that same-query decomposition into a cross-query approximation. It samples queries from calibration traces, computes their exact prefix-attention states, clusters similar queries, and stores a representative state for each cluster. A new query uses the state of its nearest cluster rather than attending to the prefix itself.

The resulting system is best understood as a **query-indexed attention surrogate**. The prefix is neither retained verbatim in the live KV cache nor distilled into model parameters. Instead, its effect on attention is tabulated over a finite query-state vocabulary. Exact online-softmax composition remains inside the merge operator, but selecting a centroid for an unseen query is approximate. The paper occasionally places these two facts too close together; the merge is exact for a correct state, while the memory as a whole is not lossless.

### Bottom-line judgment

The method is technically coherent, training-free in the narrow sense that model weights are not optimized, and experimentally promising for fixed prefixes reused across many requests. It reports favorable average ManyICLBench behavior at 1K–8K entry budgets, a peak 25.5% exact-match score on RuleArena NBA versus 24.1% with the full rulebook, nearly full-prefix banking77 performance from chunked forward passes, and sublinear lookup behavior with an optimized hierarchical search. The appendix extends the result to Qwen3-8B and Llama-3.2-3B.

The evidence remains exploratory. Most central accuracy comparisons are single-model, task-tuned sweeps that report the best validation configuration; several curves lack tabulated values or uncertainty; the memory-size response is visibly non-monotonic; and calibration can require thousands of prefix–trace forward passes plus K-means. More seriously for reproducibility, the canonical arXiv record names a GitHub repository as the code release, but that repository was empty when inspected.[^code] The paper therefore establishes an algorithmic idea and reported feasibility, not an independently reproducible serving result.

### Principal strengths

- The paper cleanly separates fixed-prefix preprocessing from online generation and preserves the online-softmax weighting needed to combine attention states.
- Chunked construction exploits an exact same-query composition property, reducing peak prefix-processing memory without discarding chunks.
- Evaluation covers in-context classification, reasoning-oriented tasks, rule-guided RAG, two additional model families/sizes, and a hardware latency study.
- The stated limitation—query distributions must form reusable local structure—is the correct central boundary.

### Principal qualifications

1. “Lossless” applies to merging exact states for the same query over disjoint blocks, not to substituting a centroid learned from different queries.
2. The method is model-, layer-, head-, prefix-, trace-distribution-, and task-configuration-specific; a changed prefix or query distribution can require reconstruction.
3. Reported O(log K) behavior depends on hierarchical indexing and a tested top-16 search policy, not on the basic flat algorithm.
4. The official code claim was not operationally satisfied by the empty linked repository at review time.

## 1. The problem the paper is actually solving

### 1.1 Problem and stakes

Many deployed LLM workflows reuse a long fixed prefix across requests: demonstrations, an instruction manual, a rulebook, retrieved documents, or an agent policy. Ordinary KV caching avoids recomputing the prefix during prefill, but every decoded token still reads prefix keys and values. Memory traffic and attention work therefore grow with prefix length. The authors also motivate “prefix decay”: as a sequence grows, a fixed prefix may receive less attention relative to accumulated generated tokens.

The paper asks whether the *effect* of a fixed prefix can be externalized so that the model processes only request and generated tokens online. This is distinct from compressing the prefix into fewer tokens, pruning KV entries, or fine-tuning weights. The target is repeated-query amortization: spend forward-only calibration work once, then reuse a finite memory bank across requests drawn from a related distribution.

### 1.2 Prior method families

**Paper report:** Prefix internalization methods use context distillation or hypernetworks to move behavior into weights or adapters. They remove prefix attention but require training and complicate updates. Prompt and KV-cache compression keep a shorter prefix in attention, preserving query flexibility but retaining online cost and possible attention dilution. Prefix caching amortizes prefill but preserves decode-time KV traffic.

The paper inherits two important ingredients. Online-softmax/FlashAttention work supplies exact blockwise attention composition. Vector retrieval and K-means supply query approximation. Its novelty is using clustered pairs of attention output and partition mass as a reusable prefix surrogate across queries.

### 1.3 The proposed gap

The claimed gap is a method that simultaneously offers training-free construction, no auxiliary trained encoder, and online work not linear in the original prefix length. This combination is supported as a design goal, but the phrase “prefix-length independent” needs care: calibration still reads the prefix, memory configuration depends on the prefix, and lookup cost depends on K. Online decode no longer scans L prefix entries; it is not globally independent of prefix preparation or chosen memory size.

## 2. Formal and technical reconstruction

### 2.1 Definitions and assumptions

For one attention head, let query (q\in\mathbb{R}^{d_h}) attend to keys and values partitioned into blocks ((K_b,V_b)). Define (a_b(q)) as normalized attention over block (b), and (Z_b(q)=\sum_{k\in K_b}\exp(qk/\sqrt{d_h})) as the block’s unnormalized softmax mass. The method assumes that queries observed at deployment are close, in a chosen representation, to calibration queries whose prefix-attention states have been computed.

Each transformer layer stores K centroids. A centroid includes a representative query, a per-head attention output, and a per-head partition mass. At inference, every query token performs a per-layer nearest-centroid lookup. Grouped-query attention requires a grouping strategy because several query heads share each KV head.

### 2.2 Block attention and merge (Equations 1–3)

Equation 1 decomposes attention over concatenated blocks:

\[
\operatorname{attn}(q,K,V)=\sum_b \alpha_b a_b(q),\qquad
\alpha_b=\frac{Z_b(q)}{\sum_j Z_j(q)}.
\]

Equations 2–3 define an associative merge. Two states ((a_A,Z_A)) and ((a_B,Z_B)) for the **same query** and disjoint blocks combine as

\[
\operatorname{merge}((a_A,Z_A),(a_B,Z_B))=
\left(\frac{Z_Aa_A+Z_Ba_B}{Z_A+Z_B},,Z_A+Z_B\right).
\]

This exactly recovers attention over concatenated blocks, aside from ordinary floating-point error. It justifies chunking a long prefix and later merging chunk states. It does not justify reusing the state of a different query; that step introduces centroid approximation error.

### 2.3 Memory bank and centroid aggregation (Equations 4–5)

At layer (i), the bank contains triples ((\bar q_k^{(i)},\bar a_k^{(i)},\bar Z_k^{(i)})). Calibration concatenates prefixes with response traces, collects each trace token’s query and exact attention state over the prefix, clusters queries with K-means, and aggregates states inside each cluster.

Equation 5 uses an arithmetic mean for query vectors and partition masses, while attention outputs are weighted by partition mass. Normalizing the centroid mass by cluster size avoids scaling a centroid as if every calibration member were simultaneously present. The formula is directionally consistent with the merge identity, but it is an approximation to a conditional expectation, not a proof that the centroid represents every member.

The HTML rendering exposes a minor tuple-ordering ambiguity in summation subscripts: prose and Equation 4 present ((q,a,Z)), while the aggregation notation appears to range over ((q,Z,a)) in places. The intended calculation is nevertheless clear from the numerator: attention outputs are weighted by their corresponding Z values.

### 2.4 Retrieval, whitening, and online merge (Equations 6–8)

Equation 6 chooses the centroid with greatest cosine similarity. The paper considers pre-RoPE keys, a common-position RoPE transform, and optional per-layer/head whitening (q\leftarrow\Sigma^{-1/2}q). Whitening tries to prevent high-variance query dimensions from dominating cosine distance.

Equation 7 combines the retrieved prefix state with the exact attention state over online, non-prefix tokens:

\[
a_{\mathrm{merge}}(q)=
\frac{Z(q)}{Z(q)+\bar Z_c}a(q)+
\frac{\bar Z_c}{Z(q)+\bar Z_c}\bar a_c.
\]

The sign and normalization are correct: weights are nonnegative and sum to one. The principal error comes from using ((\bar a_c,\bar Z_c)) rather than the unseen query’s true prefix state.

### 2.5 GQA and memory accounting

For GQA, the paper concatenates or groups query-head representations associated with a shared KV head and considers individual versus shared centroids. It sets lookup-key dimension (d'=2d_h) so that one memory entry is compared with one ordinary KV entry under its traffic accounting. The paper’s formulas compare query/output traffic plus stored prefix or lookup-key traffic. This is a model of bytes moved, not a complete peak-memory or wall-clock model: centroid states, index structures, kernel workspace, and calibration storage also matter.

## 3. Architecture and information flow

### 3.1 Offline construction plane

1. Choose a fixed prefix set and representative response traces.
2. Run forward passes over each prefix–trace pair.
3. At every layer and trace token, record the query and prefix-only attention output/mass.
4. Optionally whiten query representations.
5. Cluster queries per layer/group with K-means.
6. Compute query centroids, mean Z values, and Z-weighted attention-output centroids.
7. Build a flat or hierarchical nearest-neighbor index.

No gradients update model weights, but construction is not free: the NBA setup uses up to 6,000 iterations, and the longest construction is reported at roughly 1.5 hours on one H100.

### 3.2 Online generation plane

The online prompt omits the fixed prefix. For each layer and query token, the system retrieves one memory entry, computes ordinary attention over user/generated tokens, merges the two states by Equation 7, and continues through the transformer block. A hierarchical index expands the top 16 first-level centroids in the selected configuration.

### 3.3 Dependencies and post-processing

The system depends on an unmodified target checkpoint, custom attention integration, K-means, per-task validation selection, optional whitening, and a custom Triton lookup kernel for the reported latency study. There is no correctness validator or raw-prefix fallback in the paper. Memory contents must be versioned with the model, tokenizer, RoPE configuration, prefix, trace set, and centroid hyperparameters.

## 4. The complete operational pipeline

### 4.1 End-to-end flow

The algorithm converts a deterministic function of query and fixed prefix into a learned lookup table. Exact states are collected at sampled queries, compressed into centroids, retrieved approximately for unseen queries, and then combined exactly with live self-attention. This distinction—approximate retrieval followed by exact composition—is the correct operational description.

### 4.2 Resource allocation

The main budget is K entries per layer, swept from 1K to 16K. Increasing K improves representational granularity but requires more samples to populate clusters and more lookup work. The paper’s results show that larger is not uniformly better. Small training pools make 16K codebooks underdetermined, while the NBA task peaks at 4K.

### 4.3 Guarantee boundary

Only same-query block composition is exact. K-means approximation, nearest-centroid retrieval, finite calibration coverage, whitening, GQA grouping, and hierarchical search are empirical. There is no bound connecting cosine distance to attention-output error or downstream accuracy.

## 5. Re-conceptualization

### 5.1 Query-indexed memoization of an internal operator

**Reviewer inference:** ASM memoizes a hidden operator (f_i(q,prefix)) at every layer. Rather than cache by exact input, it vector-quantizes the query space and returns a prototype output. This is closer to approximate function memoization than to human memory or parametric learning.

**Boundary:** Classical memoization returns the exact value for a repeated key. Here, most deployment queries were never observed and receive a prototype; prefix knowledge is encoded only through the output statistics captured by calibration traces.

### 5.2 Compiled prefix behavior

The prefix is “compiled” into an index specialized to a request distribution. Compilation is forward-only and refreshable, but it is still target-specific. A compiled rulebook for one checkpoint or request family cannot be assumed to transfer.

### 5.3 Transferable design principle

When a costly static input is reused, cache sufficient intermediate statistics rather than raw input features, then explicitly preserve the normalization needed to compose cached and live computation. Test approximation at the interface where exact sufficiency becomes distributional reuse.

## 6. Experimental design reconstructed

### 6.1 Data

The ICL evaluation uses seven ManyICLBench tasks: five tasks chosen for many-shot gains plus math_counting and gpqa_cot. Memories are built from task training splits and a 32K-token prefix. RuleArena NBA uses a roughly 20K-token rulebook and synthetic training data, filtered against the test set. The paper excludes other RuleArena domains because baselines were near zero, which narrows generality and introduces task-selection conditioning.[^rulearena]

### 6.2 Models and hardware

The main model is Llama-3.1-8B-Instruct. Appendix B adds Qwen3-8B and Llama-3.2-3B. Accuracy experiments run on one H100. Latency uses one NVIDIA RTX Ada 4500, batch size 1, 512 question tokens, 100 generated tokens, and 1K–16K budgets; standard attention uses FlashAttention and memory lookup a custom Triton kernel. The main latency prose also mentions averaging an attention block over 512 generated tokens, creating a presentation inconsistency with Appendix A’s 100-token generation setting that should be resolved in reproduction.

### 6.3 Baselines

Baselines are full-context ICL/RAG, no-context, and KVzip on five ICL tasks. The paper omits prompt compression because it works at token rather than KV granularity. Uniform per-layer entry budgets improve comparability, but the method sweeps four key modes and two centroid organizations and reports the best validation configuration by task; equivalent tuning breadth for baselines is not established.

### 6.4 Metrics

Task accuracy and NBA exact match are higher-is-better. Memory comparisons equate one ASM entry with one KV entry under (d'=2d_h).[^metric] Latency is normalized to the slowest attention baseline. No confidence intervals, repeated calibration seeds, K-means initialization variance, or statistical tests are reported.

### 6.5 Runtime semantics

The latency result isolates attention rather than full end-to-end generation. It omits memory construction and index-build amortization, host/device transfer, batching above one, and service concurrency. Reported speedup should therefore be read as a kernel-path result under the specified setup.

## 7. Results: what is reported and what it means

### 7.1 ManyICLBench

Figure 3 reports that ASM matches or exceeds same-budget ICL on average from 1K through 8K entries across five selected tasks, while KVzip generally trails. At 16K, results cross over; the authors attribute this to insufficient samples for large codebooks, particularly bbh_geometric_shapes with 150 training examples. Exact figure values are not tabulated and are not estimated here.

Table 1 shows that reasoning tasks do not improve consistently with either longer ICL or larger ASM:

| Entries | math_counting ICL | math_counting ASM | gpqa_cot ICL | gpqa_cot ASM |
|---:|---:|---:|---:|---:|
| 1K | 23.9 | 24.0 | 28.3 | 28.3 |
| 2K | 22.0 | 23.0 | 28.3 | 24.2 |
| 4K | 25.0 | 22.5 | 27.3 | 27.8 |
| 8K | 26.0 | 24.5 | 24.2 | 23.7 |
| 16K | 22.5 | 26.0 | 22.7 | 23.7 |

The correct interpretation is parity and instability, not uniform superiority. This accords with the official ManyICLBench paper, which distinguishes similar-sample retrieval tasks from tasks requiring global use and reports that reasoning tasks do not necessarily improve with more demonstrations.[^manyicl]

### 7.2 RuleArena NBA

Table 2 reports zero-shot 21.2, full 20K-rulebook ICL 24.1, and ASM scores of 19.4, 19.9, 25.5, 23.2, and 19.4 at 1K, 2K, 4K, 8K, and 16K. The best 4K setting exceeds full ICL by 1.4 points at 20% of the entry-count footprint. The curve is strongly non-monotonic; selecting 4K after validation is essential to the headline.

RuleArena itself is a hard benchmark with rule-selection and computation failure modes. A small exact-match improvement does not show that rules are faithfully represented, and the result needs paired error analysis across required-rule recall, arithmetic, and final correctness.

### 7.3 Chunked construction and latency

Table 3 compares banking77 construction: a single 4K chunk with 5,000 iterations scores 61.5; a single 16K prefix with 1,250 iterations scores 79.0; merging four independent 4K chunks over 5,000 total iterations scores 78.5. This strongly supports the compositional construction claim, though iteration and token-exposure accounting should be reproduced carefully.

Hierarchical lookup with top-m at least 16 approximately matches flat top-1 accuracy across most tested first-level configurations. The optimized method is reported faster than full attention near 4K entries and 1.8× faster at 16K; the abstract gives 1.36× at 8K. Because the graph uses normalized attention-block latency on one GPU, neither value is a universal end-to-end serving speedup.

### 7.4 Cross-model appendix

On banking77, Table 5 reports Qwen3-8B ICL/ASM at 1K–16K as 50.0/32.0, 50.0/57.0, 66.0/71.5, 82.0/81.0, and 89.5/83.5. Llama-3.2-3B reports 39.5/48.0, 44.0/67.5, 60.5/77.5, 76.0/84.0, and 85.5/85.5. The claim that ASM “matches or exceeds” same-budget ICL for both backbones is not literally true for Qwen at 1K, 8K, and 16K; this is a material prose/table mismatch.

For NBA, Qwen zero-shot/full ICL are 30.6/29.6, and cache scores at 1K–16K are 31.5, 27.8, 31.0, 26.9, 25.9. Llama-3.2-3B zero-shot/full ICL are both 26.9, and cache scores are 30.1, 31.5, 29.2, 31.5, 33.3. Qwen again contradicts the prose claim of consistent superiority; only selected budgets beat both references.

## 8. Ablations and causal evidence

### 8.1 Lookup-key configuration

Table 4 shows task-specific selection among pre-RoPE, common-position/post-RoPE, whitening, and shared/individual centroid strategies. Pre-RoPE plus individual centroids is common; gpqa_cot and NBA use whitening. This demonstrates configuration sensitivity but does not isolate each factor with a common cross-task table.

### 8.2 Memory size and hierarchy

Budget sweeps are the main ablation. Non-monotonic accuracy establishes that K is not merely a fidelity knob. Figure 4 varies hierarchy breadth and first-level clusters, supporting top-16 as an empirical operating point. A missing ablation is exact attention-state error versus cosine distance; without it, the proposed mechanism is inferred from task scores.

### 8.3 Randomness and statistical evidence

K-means initialization and calibration sampling are inherently stochastic, yet no multi-seed distribution is reported. There are no intervals, paired tests, or effect sizes. Small improvements such as 25.5 versus 24.1 should be considered promising but uncertain.

## 10. External views and field context

### 10.1 Online-softmax predecessor evidence

The paper correctly grounds its merge in blockwise online-softmax computation used by memory-efficient attention. The new step is cross-query reuse through vector quantization, not the block merge itself.

### 10.2 Benchmark perspective

The canonical [ManyICLBench paper](https://arxiv.org/abs/2411.07130) reports that many-shot tasks vary between similar-sample retrieval and global context use; reasoning does not reliably improve with longer demonstrations. This limits any inference that prefix-state memory captures arbitrary long-context reasoning. The official [RuleArena ACL record](https://aclanthology.org/2025.acl-long.27/) describes difficult rule selection and computation, so exact match alone cannot diagnose faithful rule storage.

### 10.3 Contemporaneous alternatives

KV eviction and learned context compression retain a query-visible compressed prefix. Context distillation moves behavior into parameters. ASM occupies a distinct point: forward-only, distribution-specific output prototypes. Reconstructive or pageable memories would trade some online cost for recovery when a query falls outside learned clusters.

### 10.4 Publication, code, and reproducibility status

The canonical source is arXiv:2605.18226v1 with no separate proceedings record identified. The paper links [yasu0001/AttentionMemory](https://github.com/yasu0001/AttentionMemory), but GitHub displayed “This repository is empty” during external inspection. No license, dependencies, release, checkpoint, scripts, or kernel implementation could therefore be audited. Artifact availability is not verified despite the paper’s statement.[^code]

## 11. Research notes and critical considerations

### 11.1 Internal consistency

The strongest inconsistency is Appendix B’s prose versus its Qwen tables. Table 5 has ASM below ICL at three budgets, and Table 6 has cache below both references at several budgets. The latency generation length also differs between main prose and Appendix A. These should be corrected or documented in a revision.

### 11.2 Metric and baseline concerns

Best-validation configuration reporting creates an oracle-selection advantage unless deployment validation is budgeted. Equal entry counts are not necessarily equal total bytes once centroid states, keys, hierarchy, and kernel workspace are counted. Full-context attention and custom lookup also use different kernel paths.

### 11.3 Unsupported or overbroad claims

“Lossless recovery via lookup” is overstated for unseen queries. Lookup retrieves an approximate centroid; only merging the correct same-query states is lossless. “Generalizes across model family” is directionally plausible but too broad for three checkpoints and mixed Qwen outcomes.

### 11.4 Alternative explanations

ASM may outperform short ICL because each centroid has seen the full prefix and acts as supervised task adaptation through trace selection, not because attention states are intrinsically superior memory units. Validation-selected whitening and grouping may account for part of the gain. The experiments do not compare a learned regression or simpler class prototype at equal calibration cost.

### 11.5 Unresolved questions

- How does prefix-state approximation error relate to cosine distance and task failure?
- How many traces are required per effective centroid, and how stable are K-means results?
- Does the memory preserve rare rules or only modes frequent in calibration traces?
- What is end-to-end latency including lookup, merge, and all transformer layers under batching?
- How is a changed prefix updated without replaying earlier response traces?

## 12. Potential implications

### 12.1 Scientific implications

Attention output plus normalization mass is a useful interface for separating static and dynamic context. The paper exposes a broader research direction: approximate an internal operator at a composable boundary rather than compressing its raw inputs.

### 12.2 System-design implications

Serving systems could maintain versioned prefix memories for stable tenants, policies, or corpora. A routing gate should decide whether a request is in-distribution; otherwise the system should use raw prefix attention or a reconstructive memory.

### 12.3 Broader implications

The idea can transfer to other repeated static computations—retrieval reranking, multimodal encoders, or tool-policy prefixes—provided the cached statistic is sufficient for an exact downstream merge and approximation error is measurable.

## 13. New hypotheses derived from the paper

### Hypothesis 1: centroid occupancy predicts downstream reliability

**Proposition:** Sparse or high-variance clusters will produce larger attention-state and task errors.

**Motivating evidence:** Performance deteriorates at oversized codebooks and limited training pools.

**Predicted observation:** Failure probability increases as cluster occupancy falls or within-cluster state variance rises.

**Falsifying observation:** Occupancy/variance has no association with paired full-prefix error.

**Minimum test:** Archive per-query centroid occupancy, attention-output error, and task correctness across K and seeds.

### Hypothesis 2: a confidence-gated raw-prefix fallback dominates fixed K

**Proposition:** Falling back to ordinary prefix attention for low-similarity queries will improve tail correctness at lower average cost than uniformly enlarging K.

**Motivating evidence:** The limitation section identifies diffuse query distributions and prefix drift.

**Predicted observation:** A similarity/variance gate captures a disproportionate share of failures.

**Falsifying observation:** Confidence measures cannot separate failures or fallback cost erases gains.

**Minimum test:** Compare fixed ASM, larger K, and gated FullKV under equal mean latency and memory.

### Hypothesis 3: supervised trace choice is more important than centroid count

**Proposition:** Diverse, task-covering traces at moderate K outperform larger K built from narrow traces.

**Predicted observation:** Coverage-stratified trace sampling improves rare-class/rule performance without more entries.

**Falsifying observation:** Entry count alone explains performance after controlling compute.

**Minimum test:** Cross trace-diversity and K in a factorial experiment with fixed prefix tokens and forward passes.

## 14. Deployment and governance considerations

### 14.1 Appropriate use

Use only for stable, frequently reused prefixes and request families with measurable query locality, where average performance trade-offs are acceptable and a fallback exists.

### 14.2 Poor fit and failure modes

Poor fits include safety-critical rule application, rare exceptions, rapidly changing prefixes, open-domain conversations, adversarial queries, and tasks where the prefix must be quoted or audited exactly. A centroid can silently replace a rare rule with an average behavior.

### 14.3 Required safeguards

1. Version memory with model, tokenizer, prefix hash, trace set, whitening, grouping, K, hierarchy, and code commit.
2. Monitor nearest-centroid similarity, occupancy, state variance, and task-level drift.
3. Provide raw-prefix or retrieval fallback for low-confidence and high-risk requests.
4. Retain source provenance outside the memory so outputs can be audited.
5. Rebuild after prefix or checkpoint changes and canary against full attention.

## 15. Replication and falsification agenda

### 15.1 Highest-priority unresolved result

Release the implementation and reproduce Table 2 across multiple K-means/calibration seeds. Archive paired NBA predictions, rule recall, computation correctness, exact match, confidence intervals, and centroid statistics. A robust 4K advantage should persist without post-hoc K selection.

### 15.2 Reproduce the full pipeline

Pin model/tokenizer revisions; expose prefix and trace construction; implement exact per-head state collection; verify Equations 1–3 numerically; archive whitening matrices, centroids, hierarchy, and Triton kernel; and compare raw attention outputs before task evaluation.

### 15.3 Normalize evaluation

Compare total accelerator bytes and end-to-end latency rather than entry counts and attention-block time alone. Include calibration/index cost amortized over request counts, batch sizes above one, prompt changes, and common kernel/compiler versions.

### 15.4 Test the proposed mechanism directly

Measure cosine distance, true prefix-state error, merged hidden-state error, logit KL, and correctness for every query. Compare K-means with nearest-neighbor, regression, product quantization, and learned but lightweight routers.

### 15.5 Test realistic shifts and alternatives

Calibrate on one time period or domain and test another; inject rare rules; change the prefix incrementally; use long multi-turn chat; test hostile centroid-attracting prompts; and compare recoverable/pageable prefix memory.

## 16. Durable restatement

Attention-State Memory replaces repeated reads of a static prefix with an approximate per-layer function table. Calibration records how representative queries attend to the prefix, clusters those observations, and stores prototype outputs plus the softmax mass needed to combine them with live self-attention. The exact blockwise merge is well founded; cross-query reuse is the empirical approximation. Reported results show that this can be useful for stable in-context and rulebook workloads, especially at intermediate budgets, but non-monotonic accuracy, task-specific tuning, internal result mismatches, and an empty code repository keep the evidence below reproduction grade.

What should endure is the interface design: cache a sufficient statistic that composes correctly, then treat the approximation used to retrieve that statistic as the scientific object that must be measured, monitored, and allowed to fail safely.
