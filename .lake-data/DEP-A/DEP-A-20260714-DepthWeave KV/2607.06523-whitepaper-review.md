# Depth-Adaptive KV Compression Without a Reproducible Experimental Identity

## A full-paper review and independent re-conceptualization of “DepthWeave-KV: Token-Adaptive Cross-Layer Residual Factorization for Long-Context KV Cache Compression”

**Paper:** Anna Cordoba, Adam Puente Tercero, Nerea Angulo Hijo, Mar Linares Tercero, Julia Barrientos, Ainhoa Miranda, and Jesus Olivera, arXiv:2607.06523v1, 7 July 2026.[^1]  
**Scope:** complete PDF, full-paper HTML, TeX source, all sections, two numbered tables, two figures, three numbered equations, references, and associated repository records.  
**Review activity:** full-source reconstruction and consistency audit only. No experiment or implementation was reproduced.

---

## Executive assessment

### The paper in one sentence

DepthWeave-KV proposes shared low-rank bases across adjacent transformer layers plus token-routed residuals and online attention-error probes, but its exceptionally favorable results cannot be evaluated because the paper omits the base model, hardware, training recipe, code, and statistical protocol, while its TeX explicitly identifies the author block as synthetic.

### Technical promise

The underlying design is plausible. Adjacent transformer layers often contain redundant KV structure; a shared basis can capture the common component, and residual capacity can be spent selectively on tokens whose attention behavior is fragile. Fusing reconstruction with attention is also the correct systems instinct because materializing a full reconstructed cache would surrender much of the bandwidth benefit. The paper combines three useful levers: depth sharing, token-adaptive fidelity, and output-space error control.

### Bottom-line judgment

The artifact does not support its empirical claims at whitepaper-grade confidence. Table 1 reports eleven methods across seven metrics, with DepthWeave-KV winning every compressed-method metric: 62.9 average score, 96.1% Needle accuracy, 8.3× memory reduction, 72.8 tokens/s, 4.89 seconds TTFT, +0.09 perplexity, and 0.027 reconstruction error. Yet no base model is named. No accelerator, precision, batch size, software stack, random seed, uncertainty estimate, router training corpus, kernel repository, or code URL is provided. Throughput and TTFT are meaningless without those conditions.

The source provenance is more concerning. The manuscript TeX source uses accepted ICML 2024 camera-ready mode for a paper submitted to arXiv in July 2026. A source comment says “Fixed synthetic byline,” all authors share a generic artificial-vision institute, and the corresponding address uses the unusual `.tibeu` domain. These are source-level facts, not speculation about the people. They require the review to treat all identity, venue, and result claims as unverified beyond the existence of the arXiv record.

There are also internal method contradictions. The abstract says calibration-free and no base-model retraining, but the method later says residual gates and router parameters are trained on unlabeled long-context text. Equation 2 specifies a binary threshold producing minimum or maximum rank, while the prose says three rank levels are used. Online probes compare compressed attention with a temporarily materialized higher-fidelity cache, but the paper never explains how that reference cache remains available after aggressive compression or how its memory and compute are charged. The residual is a full `1×d` vector “stored only at the routed rank,” an undefined mapping between vector channels and rank.

The appropriate conclusion is therefore bifurcated: the architecture is a worthwhile hypothesis; the reported implementation and results are not independently assessable and should not guide deployment until code, model identity, hardware, raw outputs, and provenance are supplied.[^5] Every use of “near-full-cache” below remains explicitly paper-reported.[^6]

## 1. Problem framing and actual novelty

Long-context decoding stores one key and value per retained token, layer, and attention head. This state grows linearly with context length and often limits batch size or maximum sequence length. Existing methods attack different axes: token eviction removes positions, quantization reduces bits, low-rank projection reduces channel dimension, and layer sharing reuses states across depth.

The paper’s novelty claim is the joint use of three axes:

1. **Cross-depth residual factorization:** neighboring layers share per-head key and value bases.
2. **Token-conditional residual rank:** tokens judged salient receive more layer-specific correction.
3. **Online output-error feedback:** periodic probes change the rank threshold during generation.

MiniCache is the closest inherited component because it explicitly compresses in the depth dimension.[^2] DepthWeave-KV changes hard or discrete sharing into a factorized shared component plus residual. Token salience resembles heavy-hitter and structural-retention methods. Attention-output reconstruction resembles approaches that optimize downstream attention behavior rather than raw vector error. The fused kernel is an implementation claim needed to turn algebraic compression into actual bandwidth savings.

The conceptual novelty lies in their composition and routing granularity, not in discovering depth redundancy, low-rank bases, salience scoring, quantized residuals, or fused attention individually. A faithful evaluation must separate gains from each inherited component, which Table 2 attempts but cannot validate without executable artifacts.

## 2. Technical reconstruction of the method

### 2.1 Shared depth-window bases

Layers are partitioned into overlapping windows `Wm` of size `w`. For each window and attention head, the method stores shared key and value basis matrices `Bᶻm,h ∈ R^(rb×d)`. A token’s reconstructed key or value at layer `ℓ` is:

`X̂ᶻℓ,h,t = Aᶻℓ,h,t Bᶻm,h + gᶻℓ,h,t Rᶻℓ,h,t`, for `Z ∈ {K,V}`.

`A` is a `1×rb` coefficient vector. `R` is described as a `1×d` residual. `g` is a scalar gate in `[0,1]`. The shared basis is amortized over tokens and layers in a window; coefficients remain token- and layer-specific. If most variance is shared across adjacent layers, storage can fall while reconstructing distinct layer states.

The basis is reportedly initialized from a randomized low-rank sketch of the first compressed prefix block and updated by exponential moving average during generation. Keys and values, heads, and windows are handled separately. Coefficients use 8-bit blockwise quantization; residuals use 4-bit group quantization.

Several necessary details are absent: exact sketch algorithm, `rb`, window size and overlap, EMA factor, update frequency, quantization group size, scale/zero-point representation, coefficient overhead, basis overhead, residual indexing, and numerical accumulator types. Without these, the 8.3× accounting cannot be reconstructed.

### 2.2 Token-conditional router

The router uses accumulated attention mass, maximum attention spike, delimiter or instruction-boundary indicators, and prior probe error. Equation 2 sets:

`ρ = ρmin + (ρmax - ρmin) · 1[σ(uᵀφ) > τm]`.

This is a two-level decision. The next paragraph says the implementation uses three levels: typical continuation tokens receive 0 or 2, structural tokens 4, and critical spans 8. The paper does not provide the multiclass or ordinal rule that produces those values. It also says `u` is a learned router parameter, but supplies no architecture, loss, optimizer, data, schedule, or checkpoint.

Threshold `τm` is adjusted to meet memory budget. A complete controller would need to explain whether ranks are promoted and demoted for already stored tokens, how demotion affects residual data, whether selection is per head or token, and how global budget is enforced across windows. None is specified.

### 2.3 Attention-output probes

Every `p` decoding steps, a subset of heads and recent queries is evaluated with compressed and “temporarily materialized higher-fidelity” cache states. Equation 3 computes relative L2 attention-output error:

`e = ||Attn(Q,K,V) - Attn(Q,K̂,V̂)||₂ / (||Attn(Q,K,V)||₂ + ε)`.

If error is high, the threshold falls and more tokens receive residual capacity; if low for several intervals, compression increases. Output-space error is a better proxy than uniform KV MSE because attention is sensitive to directions and query alignment.

The reference path is undefined. If original `K,V` have been discarded, exact attention cannot be computed later without retaining or recomputing them. Retaining them defeats cache savings; recomputing them costs model execution. If only recent queries or a small high-fidelity shadow are retained, the probe estimates a restricted error and must charge shadow storage. The paper reports neither probe interval, sample size, shadow-cache size, duplicated attention cost, nor its share of throughput.

### 2.4 Residual rank ambiguity

The paper repeatedly speaks of residual “rank” and “retained residual channels,” yet `R` is a single `1×d` vector per token/layer/head. Rank is not defined for one vector in the usual low-rank sense. It may mean number of quantized residual basis channels, selected dimensions, or codebook components, but the reconstruction equation shows none of those. This gap prevents implementation and storage analysis.

### 2.5 Fused execution

The claimed CUDA kernel loads bases, dequantizes routed residuals, multiplies coefficients, and immediately consumes the reconstruction in attention without writing a full KV tensor to global memory. That pipeline is architecturally sound. However, no tiling, supported head dimension, memory layout, kernel launch configuration, parallelization strategy, GPU target, or code is given. The reported 72.8 tokens/s and 15% ablation gain cannot be connected to an implementation.

## 3. Architecture, training, and inference pipeline

The paper describes prefill as four stages: create depth-window bases, project each KV vector to coefficients, route residual fidelity, and record probe statistics. During decode, bases and coefficients are loaded, residuals dequantized, reconstructed fragments consumed by attention, and occasional probes update thresholds.

It claims base-model weights are unchanged. That is different from requiring no training at all. The final method paragraph states that residual gates and router parameters are trained once on unlabeled long-context text against a frozen teacher cache. Therefore the correct label is **base-model frozen, auxiliary compressor trained**, not calibration-free in the ordinary sense. “Calibration-free” may mean no task-specific post-training calibration, but the paper should say so.

There is no model architecture, layer count, head layout, MQA/GQA handling, positional-encoding treatment, training corpus, teacher objective, training duration, optimizer, or released parameter file. Whether one router transfers across models is unknown. The method cannot currently be reconstructed from the paper.

## 4. Experimental design and evidence reconstructed

The benchmark list includes LongBench, L-Eval, NarrativeQA, Qasper, HotpotQA, MultiFieldQA-en, GovReport, QMSum, TriviaQA, and Needle-in-a-Haystack. The main setup says 64K prefill and 512 greedy decoding tokens. Needle results allegedly average 16K–128K context lengths and depths 0–100%. The average task score is normalized over unspecified subsets.

Baselines are Full KV Cache, StreamingLLM, H2O, SnapKV, PyramidKV, MiniCache, KVSharer, ChunkKV, TailorKV, and Eigen Attention. Compression methods target their strongest score at at least 4× memory reduction. This tuning rule is not sufficient: baseline versions, repositories, configurations, observation windows, budgets, and model compatibility are missing.

The base model is never named. “All methods use the same base model” is not a model identifier. Hardware is never named. Tokens per second and TTFT depend on GPU, CUDA, clocks, memory, batch, sequence layout, prompt caching, and measurement boundaries. No seeds, number of examples, confidence intervals, or significance tests appear. No dataset licenses or exact evaluation scripts are stated.

This is not merely incomplete reproducibility documentation. It prevents basic semantic interpretation. An average score of 62.9 could be excellent or ordinary depending on model and task weighting. A 72.8 tok/s decoder could be slow or fast depending on GPU and batch. A 4.89-second TTFT for 64K depends on whether prefill is included and what hardware executes it.

## 5. Results reconstruction

Table 1 reports the following for DepthWeave-KV: 62.9 average score, 96.1 Needle accuracy, 8.3× KV reduction, 72.8 tok/s, 4.89 s TTFT, +0.09 perplexity, and 0.027 reconstruction error. Full cache is 63.8, 98.7, 1×, 42.1 tok/s, 6.84 s, zero perplexity delta, and zero reconstruction error.

TailorKV is the strongest compressed baseline by reported average score at 61.4 and Needle at 92.6. It reports 6.5× reduction, 65.0 tok/s, 5.36 s TTFT, +0.18 perplexity, and 0.049 reconstruction error. DepthWeave-KV therefore claims to dominate it on every column.

The remaining baseline scores are internally ordered in a smooth progression: StreamingLLM 55.4/79.6; H2O 57.1/83.8; SnapKV 58.3/86.5; PyramidKV 59.0/88.1; MiniCache 59.8/89.4; KVSharer 59.5/88.7; ChunkKV 60.7/91.2; Eigen Attention 60.2/90.1. Such results are possible, but without raw task-level outputs, exact configs, and uncertainty they are only paper-reported numbers.

The prose adds family aggregates: DepthWeave-KV trails full cache by 1.4 points on extractive/retrieval tasks versus 5.6 for MiniCache and 3.2 for TailorKV. On summarization it leads TailorKV by 0.8 and ChunkKV by 1.7. The underlying per-task table is absent, so these aggregates cannot be audited for denominators or weighting.

## 6. Ablations audit

Table 2 reports seven variants. Removing cross-depth sharing gives 61.5 average, 93.2 Needle, 5.9× reduction, 63.5 tok/s, 5.31 s TTFT, +0.16 perplexity, and 0.044 error. Removing the router yields 61.2/92.4/7.9×. Removing online tracking yields 61.7/93.5/8.5×. Removing gates yields 61.0/91.8/8.4×. Shared bases without token residuals gives 58.9/87.6/10.2×. Removing the fused kernel preserves quality and memory but drops throughput from 72.8 to 61.9 and raises TTFT from 4.89 to 5.58.

These ablations are directionally coherent: bases drive memory reduction, residual/routing components protect retrieval, and fusion drives speed. But they are not enough to distinguish components rigorously. There is no fixed-memory comparison for router versus uniform ranks, no probe-overhead accounting, no basis-rank or window sweep, no threshold sensitivity, no quantization-only baseline, no training-data ablation, and no per-task variance. The paper’s prose also conflicts with earlier ablation values: the experiments section says fixed routing gives 61.1/91.8 and disabling probes gives 89.9 Needle, while Table 2 gives 61.2/92.4 and 93.5. This table/prose inconsistency is material.

## 8. External primary-source vetting

The canonical arXiv record confirms the title, seven-author list, v1 date, and abstract claims.[^1] It does not expose a code or data link. Web checks found no official DepthWeave-KV repository attributable to the paper. Search-index copies repeat the abstract but provide no independent evidence.

The MiniCache canonical record verifies that depth-dimension KV compression is prior work and provides a real conceptual baseline.[^2] It does not validate DepthWeave-KV’s implementation or results. Same-paper context exists in the Black Lake DEP-E index and Delphoa-Labs/Black-Lake-Data; those records are associated archival context, not full-paper validation.[^3]

The TeX source is primary evidence of the provenance anomaly: it includes the literal comment “Fixed synthetic byline,” accepted ICML 2024 mode, a generic affiliation, and a `.tibeu` email.[^4] This review does not infer who generated the paper or why. It records that these source facts materially weaken venue, author-affiliation, and reproducibility confidence.

## 9. Research notes and critical considerations

### 9.1 A probe needs a truth source

Adaptive error control is attractive, but the controller must compare against something. A shadow cache, recomputation path, or sampling checkpoint changes the actual rate–latency tradeoff. The method should be reformulated around explicit checkpoint storage and charged probe cost.

### 9.2 “Rank” should be replaced by a defined code

A token residual could be represented by selected residual basis coefficients. Then rank levels 0/2/4/8 would have a precise meaning, and Equation 1 would include a residual basis. The current full-vector residual does not support the stated rank routing.

### 9.3 Provenance is part of reproducibility

A paper can contain mathematically coherent text and still lack a verifiable experimental identity. Base model, hardware, code, author affiliation, and venue are not administrative trivia; they determine whether results can be tested and attributed.

### 9.4 Dominating every metric raises the evidence burden

Compression usually trades quality, memory, and compute. Claiming simultaneous improvement in quality, reduction, throughput, TTFT, perplexity, and error is not impossible, especially with a fused layout, but requires stronger evidence: Pareto curves, controlled hardware, profiler traces, and raw results.

## 10. Limitations, implications, and deployment applicability

The paper contains no dedicated limitations section. Missing limitations include auxiliary-training dependence, probe reference cost, basis drift, model transfer, GQA/MQA behavior, prompt-adaptive instability, quantization overflow, kernel portability, and irreversible reconstruction error. No funding, conflict, ethics, energy, or data-governance disclosure was located.

For research, the design motivates a useful compressor: share across depth, retain sparse token corrections, monitor output error, and fuse reconstruction. For deployment, nothing is ready. Operators cannot estimate memory, compile a kernel, obtain router weights, reproduce scores, or know supported models. Until an implementation is available, the paper should be treated as a specification sketch.

Governance also matters because token salience can preferentially preserve instructions, names, or citations. A learned router may encode workload bias and could discard qualifiers or safety constraints. A deployment must expose retention diagnostics, allow high-fidelity protected spans, and fail closed when probe confidence is poor.

## 11. Independent re-conceptualization and falsifiable hypotheses

### Hypothesis 1: residual-basis routing is implementable and superior

Replace the undefined residual vector with a learned residual basis per window and route 0/2/4/8 coefficients. At equal total bytes, this should outperform uniform coefficients on Needle and Qasper. Falsify if uniform allocation matches it.

### Hypothesis 2: probe-aware accounting materially reduces the speed claim

Charge shadow-cache bytes and duplicate attention FLOPs. Net throughput gain should be lower than 72.8 versus 42.1 but remain positive if fusion works. Falsify if full accounting removes the gain.

### Hypothesis 3: token routing helps retrieval more than summarization

At matched 8× storage, adaptive ranks should improve exact retrieval by at least two points over uniform ranks while changing summarization less than one point. Falsify if gains are uniform or absent.

### Hypothesis 4: depth windows are model-specific

Optimal window size and basis rank should differ across model families and GQA layouts. Falsify if one router and basis configuration transfers without loss across three families.

## 12. Replication and falsification agenda

1. Identify the exact base model, revision, tokenizer, and attention layout.
2. Release router/gate training code, data manifest, objective, optimizer, and weights.
3. Define residual rank algebraically and publish complete memory accounting.
4. Explain how high-fidelity probe references are retained or recomputed.
5. Release the CUDA kernel with supported hardware and profiler traces.
6. Report GPU, CUDA, precision, batch, clocks, warmup, timing windows, and peak memory.
7. Publish per-task values, prompt templates, seeds, confidence intervals, and raw outputs.
8. Resolve the Table 2 versus prose ablation discrepancies.
9. Run fixed-byte Pareto sweeps rather than one selected point per method.
10. Clarify author affiliation, venue status, synthetic-byline source comment, and public code ownership.

Failure to provide a reconstructible reference-cache path, or failure to reproduce even the full-cache baseline under named hardware and model, would falsify the current systems claims.

## 13. Durable restatement

Depth-adaptive KV compression should represent adjacent layers with shared bases, assign explicit residual coefficients to tokens according to downstream risk, and fuse reconstruction with attention. An online controller is valuable only when its truth source and overhead are included in the budget. DepthWeave-KV sketches that architecture and reports attractive numbers, but the experimental identity is missing and the source provenance is anomalous. The design deserves implementation; the reported table does not yet deserve operational trust.
