# Token-Sequence Entropy Is Not Yet a Fast KV Cache Codec

## A detailed review, technical reconstruction, and independent re-conceptualization of “Sequential KV Cache Compression via Probabilistic Language Tries: Beyond the Per-Vector Shannon Limit”

**Paper:** Gregory Magarshak, *Sequential KV Cache Compression via Probabilistic Language Tries: Beyond the Per-Vector Shannon Limit*, arXiv:2604.15356v1, 10 April 2026.[^1]  
**Canonical record:** https://arxiv.org/abs/2604.15356  
**Reviewed evidence:** locally archived full PDF and full-paper HTML; canonical arXiv record; canonical TurboQuant and PLT records; and vLLM’s exact-prefix-caching design documentation.[^2]  
**Coverage:** complete 22-page theory/proposal paper, every definition/lemma/theorem/corollary/proposition/conjecture, implementation sketch, limitations, references, and all displayed equations. The paper contains no numbered tables or figures and no empirical study.  
**Reproduction status:** no code, model, data, or benchmark artifact was identified; no experiment was reported or rerun.

---

## Executive assessment

### The paper in one sentence

The paper observes that exact KV tensors are deterministic functions of token prefixes, bounds their conditional discrete entropy by next-token entropy, and proposes cross-session prefix sharing plus within-session predictive residual coding—but it does not bridge that information bound to a practical random-access cache codec.

### The deeper idea

There is a legitimate and important distinction between coding vectors independently and coding a sequence with side information. If the decoder knows the preceding token prefix, only the next token choice is random. A fixed deterministic transformer maps that discrete choice to a full set of keys and values. In a pure source-coding sense, the new exact tensor state cannot contain more Shannon information than the next token. The paper uses this fact to contrast a per-vector distortion bound with a conditional sequence-description bound.

It then proposes two mechanisms. Across sessions, a probabilistic-language-trie (PLT) index should share exact common-prefix KV and store tail deltas relative to a centroid. Within a session, the model’s next-token distribution should induce an expected KV state; the cache stores a quantized residual between actual and expected KV. Per-vector quantization can then compress residuals.

### Bottom-line judgment

The core entropy observation is mathematically elementary but potentially clarifying: exact deterministic activations can be described by the discrete input that produced them. It does not imply that millions of KV component bits can be replaced by roughly four stored bits while retaining the purpose of a KV cache—fast reuse without recomputing the transformer. Tokens are already available. Given tokens and weights, the entire KV cache can always be omitted and recomputed; this obtains an excellent storage rate by surrendering cache speed. A useful compression result must jointly bound stored bits, reconstruction compute, access pattern, latency, and attention distortion.

Several paper-specific arguments are internally inconsistent or unsupported. Under the stated 70B example, `2×80×64×128×128,000×2` bytes is about 312.5 GiB, not approximately 80 GB. The proposed PLT quantity `-log2 P(longest common prefix)` grows as a common prefix extends because prefix probability decreases; this conflicts with the paper’s use of small distance/`≤δ` as “closer,” while its lookup section later says to maximize the same quantity. It also cannot group the paper’s paraphrase example—“You are a helpful assistant” versus “You are an AI assistant”—beyond the lexical prefix at which the strings diverge. The expected KV predictor requires evaluating the transformer under candidate next tokens, and the claimed `O(k·d_model)` overhead is not derived from the actual forward computation.

The advertised 914,000× theoretical ratio is an apples-to-oranges quotient: bits used by a direct lossy vector representation are compared with entropy of an indirect symbolic description whose decoder may need model execution to materialize the tensor. The paper candidly calls empirical residual measurement and a kernel implementation open problems, but the abstract states practical ratios far more strongly than the evidence allows.

### Principal strengths

- It correctly demands that KV compression be analyzed as a sequence rather than only as independent vectors.
- It distinguishes token conditional entropy from realized surprisal in later sections.
- It recognizes that an implementable predictor must approximate a vocabulary expectation.
- It explicitly identifies empirical tightness, kernels, and non-transformer extensions as open problems.
- It attempts to connect lossless entropy, lossy rate–distortion, prefix reuse, and quantization in one framework.
- It provides falsifiable implementation targets even though none is executed.

### Principal qualifications

- No empirical evidence supports residual size, compression ratio, latency, or quality.
- Exact source entropy is not a sufficient cache data-structure result.
- The proof’s sigma-algebra equality requires injective token-to-layer-1 KV projections, which is assumed generically rather than verified for the model.
- The PLT “metric” is called a pseudoultrametric despite nonzero self-distance and is used with contradictory directions.
- The Lipschitz constants can be enormous, making magnitude bounds vacuous; explicit formulas are deferred to a forthcoming paper.
- The monotone conditional-entropy condition is neither typical pointwise language behavior nor empirically demonstrated.

## 1. Problem and baseline

### 1.1 KV cache purpose and size

At every layer and position, a transformer computes keys and values from hidden activations. Caching them avoids recomputing earlier positions during autoregressive decoding. For `L` layers, `H_kv` KV heads, head dimension `d`, context `n`, and element width `w`, storage is `2 L H_kv d n w` bytes.

The paper writes `H_head=64`, `L=80`, `d=128`, `n=128,000`, and fp16. Those values yield 335.5 billion decimal bytes or 312.5 GiB. An 80GB result would require about sixteen KV heads or another changed assumption, as in grouped-query attention. This arithmetic error matters because the paper uses the same 64-head configuration in its bits-per-token ratios.

### 1.2 Per-vector quantization boundary

TurboQuant studies online vector quantization under distortion objectives. Its canonical record describes a random rotation, coordinate quantization, and one-bit QJL correction, with formal near-optimal distortion-rate claims and reported quality neutrality at 3.5 bits per channel.[^3] That is a direct representation: attention can operate from compressed values under a defined distortion and runtime path.

The reviewed paper instead asks how few bits identify an exact KV outcome when the prior prefix and model are side information. This is a different question. It may have a lower rate, but only if the decoder cost and required state are accounted for.

## 2. The sequential entropy result

### 2.1 Determinism lemma

For a fixed model with deterministic inference, the full KV state at position `i` is a function `F_M(t_1,…,t_i)`. There is no stochasticity after token inputs, model weights, positions, and numerical execution are fixed. Thus a random KV tensor induced by random tokens has support no larger than the token-prefix support.

This is correct at the abstract mathematical level. Real kernels can be nondeterministic due to parallel reductions, but that is an implementation issue rather than the theorem’s main limitation.

### 2.2 Conditional entropy theorem

The paper states

`H(KV_i | KV_<i) ≤ H(t_i | t_<i) = H(t_i | KV_<i)`.

The inequality from deterministic data processing is available if conditioning on the token prefix: `H(KV_i | t_<i) ≤ H(t_i | t_<i)`. To replace token-prefix conditioning with prior-KV conditioning, the paper proves that prior KV determines prior tokens. It assumes the layer-1 key projection is injective on every finite pair of token embeddings for a fixed context, arguing this holds generically with probability one.

For a concrete trained model, genericity is not a verified property. The embedding matrix cannot literally have “full column rank” when vocabulary size exceeds hidden dimension, although pairwise distinct columns are possible and sufficient for the first step. A low-dimensional key projection can still be injective on a finite vocabulary, but it must be checked; quantized or grouped implementations can introduce collisions. Without injectivity, the inequality can be retained with token prefix as decoder side information, while the claimed sigma-algebra equality may fail.

### 2.3 What the theorem actually buys

Under injectivity, the paper later observes that equality holds: exact `KV_i` and `t_i` encode the same discrete uncertainty given the prefix. This makes the “bound” less surprising. A code can store the token ID using an arithmetic coder driven by the model probability. To recover the exact KV tensor, it must run `F_M` on the decoded token and context. The result therefore describes the storage/compute frontier’s recompute endpoint, not a cache that has escaped it.

The paper’s comparison with 3 bits per component ignores that TurboQuant’s bits support approximate direct tensor use, whereas 3–4 token-level bits identify which of a finite set of huge tensors should be computed. A fair theorem would include a decoder complexity constraint or define an access model in which tensors can be reconstructed without a full forward path.

### 2.4 Perplexity and distribution mismatch

Average next-token entropy equals model cross-entropy only when the token sequence is drawn from the model’s own distribution and expectations are defined consistently. On an external data distribution, average realized negative log probability is cross-entropy, not necessarily `H_P(t_i|prefix)`. The paper alternates between sequences drawn from `P_M` and perplexity on dataset `D`. A model’s reported perplexity of 10–20 does not imply the actual KV source entropy on arbitrary user prompts is 3.3–4.3 bits unless the modeling/distribution assumptions are stated carefully.

## 3. Claimed compression ratios

### 3.1 Arithmetic reconstruction

At `L=80`, `H=64`, `d=128`, and 3 bits/component, the paper computes `2×80×64×128×3 = 3,932,160` bits per token. Dividing by 4.3 yields about 914,456. The arithmetic is correct. The fp16 numerator `20,971,520` bits/token divided by 4.3 is about 4.88 million.

### 3.2 Why the quotient is not a practical codec ratio

The numerator is a materialized tensor code optimized for attention distortion. The denominator is the entropy of a symbolic token outcome under side information that includes model and prior state. If decoder execution is free, the cache can store zero tensor bits plus tokens and recompute everything. But KV caching exists because decoder execution is not free.

The paper next assumes practical source coders are 10–20× or, pessimistically, 1,000× above entropy, yielding 45,000–91,000× or 914× over TurboQuant. Arithmetic-coder overhead is not the relevant overhead: the expensive object is reconstructing millions of activation components for fast attention. No code construction or complexity proof converts a 4.3-bit token code into a direct KV representation at 4,300 bits.

### 3.3 Required corrected claim

The defensible statement is: “Without a decoder-compute constraint, the exact KV sequence’s symbolic description can be no longer in expectation than the token sequence that generates it.” That motivates predictive coding research. It does not establish any realized KV compression ratio over TurboQuant.

## 4. Layer 1: prefix sharing and PLT audit

### 4.1 Exact prefix caching

vLLM’s design hashes blocks using exact token IDs (plus other prefix-affecting identifiers) and reuses the corresponding cache when prefixes match.[^4] Exactness ensures the reused KV is numerically appropriate for the same model/configuration. The reviewed paper correctly notes that identical system prompts can be highly reusable.

### 4.2 PLT definition and direction error

The PLT paper defines a probability-weighted prefix tree.[^5] Here, `d_T(s,s') = -log2 P_M(s∧s')`, where `s∧s'` is longest common prefix. Prefix probability is a product of conditional probabilities and ordinarily decreases or stays equal as the prefix grows. Therefore `-log probability` ordinarily increases with common-prefix length. Two strings sharing only a high-probability BOS can have a small value; identical long strings have a large nonzero self-value.

This contradicts several uses in the paper:

- §2 says long, high-probability shared prefixes are “close,” but length and probability pull the value upward and downward differently.
- Definition 3 clusters pairs satisfying `d≤δ`, selecting short/high-probability common prefixes rather than necessarily deep overlap.
- §7’s best-match operation says to maximize `d`, the opposite direction.
- The paper calls the object a pseudoultrametric, but standard pseudometrics require zero self-distance. The text itself admits `d(s,s)=-log P(s)≠0`.

### 4.3 Semantic-equivalence overclaim

The trie operates on lexical token paths. “You are a helpful assistant” and “You are an AI assistant” diverge after “You are a”; their longest common prefix is not lengthened by semantic similarity. A model probability assigned to the shared lexical prefix does not compare the meanings of divergent suffixes. The method can weight exact-prefix overlap, not deduplicate arbitrary paraphrases without an additional semantic mapping or activation-distance test.

### 4.4 Delta storage analysis

Before lexical divergence, KV states are exactly equal. After divergence, the paper proposes a norm bound using layerwise Lipschitz constant `κ^ℓ` times embedding difference. If `κ≥1`, this can grow exponentially with depth and is not evidence the tail delta is small. The full constant formula is deferred to an unpublished companion.

The relative storage formula `1-f(1-̄ℓ/n)` assumes cluster members store only tail positions and ignores centroid amortization details, delta bit widths, index/pointer metadata, and post-divergence full-tail differences. The text says `̄ℓ≪n` corresponds to sessions diverging early; a short tail after divergence actually means late divergence. “Factor-f reduction” is also imprecise when `f` is a fraction: savings approach fraction `f`, while remaining cost approaches `1-f`.

## 5. Layer 2: predictive delta coding

### 5.1 Exact expected-KV predictor

The proposed predictor is the probability-weighted mean of `F(prefix,t)` across every vocabulary token. This is the minimum-mean-square predictor under the assumed distribution. Subtracting a deterministic conditional mean is bijective, so it does not reduce discrete conditional entropy: the paper’s proof explicitly concludes equality, despite stating `≤` first.

It can reduce residual variance, which may help lossy quantization. Low token entropy alone yields a useful bound only after multiplying by a model Lipschitz constant and maximum embedding diameter. Without measured constants, the bound may be vacuous.

### 5.2 Top-k approximation cost

Computing `F(prefix,t)` for each of the top `k` tokens is not merely `O(k·d_model)`. `F` denotes all-layer KV produced by processing the candidate token with attention to prior context. Even if candidate evaluation is batched and prior cache is shared, it incurs projection, attention, MLP, and memory traffic for each candidate. The paper itself initially says it requires a forward evaluation per vocabulary token, then later collapses top-k cost to a vector-width expression without derivation.

For each actual generated token, the ordinary forward pass already reveals its exact KV. Running up to twenty additional candidate passes to predict a tensor before storing it can overwhelm decode cost. During reads, “recompute identically from prior context” can be circular: fast reconstruction of each old position may require the very prior KV entries being decoded, or sequential replay from tokens.

### 5.3 Linear approximation

The alternative linearizes each layer’s token-to-KV map as context-dependent offset plus matrix times embedding. No method is given to compute those matrices cheaply, no approximation error bound is supplied, and no experiment tests them. It is a research direction, not an implemented codec.

### 5.4 Adaptive bit allocation

The heuristic `b_i=max(1,floor(b0 h_i/̄h))` allocates more bits to surprising tokens. It does not “achieve” the entropy bound by itself; it still assigns bits per component across a huge residual tensor and lacks a scale/codebook/error specification. Realized surprisal is known after the token occurs, so it can guide allocation, but decoder synchronization requires storing or recomputing the same probabilities and bit lengths.

## 6. Composition and asymptotics

### 6.1 Orthogonality claim

Exact shared-prefix pointers and tail coding address disjoint positions by construction. That establishes non-overlap of the chosen mechanisms, not statistical independence. The paper says the gains depend on “statistically independent sources” and later softens this to different aspects not constrained to be equal. Session overlap and token surprisal can clearly correlate—for example standardized prompts are both shared and predictable.

### 6.2 Composition with quantization

Residual quantization is a plausible stack. Recovered KV equals predicted KV plus quantized residual, so tensor error equals residual quantization error if the predictor is reproduced exactly. The missing question is whether prediction cost plus residual metadata yields a better speed–memory–quality frontier than directly quantizing actual KV. No result answers it.

### 6.3 Context-length claim

The paper defines coherent context by pointwise non-increasing conditional entropy. Natural language has punctuation, entities, topic shifts, and dialogue turns; the paper acknowledges spikes but asserts the condition holds in expectation for stable documents without evidence. For a stationary source, conditional entropy can converge to a positive entropy rate rather than continue falling. Total sequential description still grows linearly at that rate. “Long-context inference becomes cheaper per marginal token” is conditional, not a general asymptotic consequence of having more context.

## 7. Rate–distortion and speculative-decoding sections

### 7.1 Rate–distortion audit

The paper upper-bounds a vector rate–distortion function by a Gaussian source with the same second moment. This can be a useful relaxation, but notation alternates between full-KV dimension, head dimension, and per-component distortion. The covariance spectrum across all layers/heads is not represented by `d_head` alone. The isotropic step is also described as an upper bound even though distributing fixed total variance equally maximizes the sum only under precisely aligned distortion accounting.

The conclusion that zero bits suffice when allowed distortion exceeds residual variance is true in a source-coding sense: reproduce the mean. It does not say attention/output distortion is acceptable. Mean-squared KV error is not directly calibrated to downstream attention logits, layer propagation, or generation quality.

### 7.2 Speculative-decoding connection

The paper observes that a top-k token distribution appears in both speculative decoding and predicted-KV averaging. But a speculative draft distribution does not automatically provide `F_M(prefix,t)` under the target model for every candidate. Candidate verification may batch target computation, yet “zero additional inference overhead” requires those target KV alternatives to be produced and retained by the same pass. The theorem shows shared probabilities, not a complete fused computation.

## 8. Results, experimental design, and ablations

There are no experiments, datasets, tables, figures, hardware measurements, baselines, seeds, or ablations. All ratios beyond arithmetic bounds are projections. The paper proposes future evaluation on Llama, Gemma, and Mistral, a Triton kernel, and measured residuals. Those are prerequisites for a systems claim, not optional extensions.

A minimum empirical study must compare: fp16/bf16 KV; TurboQuant or another direct quantizer; token-only recomputation; exact prefix caching; actual residual coding with top-k and linear predictors; and combinations. It must measure memory, prefill/decode latency, attention error, perplexity, long-context task quality, energy, and concurrency.

## 10. External primary-source vetting

### 10.1 Identity and publication status

The canonical arXiv record confirms one author, v1, 22 pages, cs.LG/cs.AI/cs.IT/cs.NE classifications, and an arXiv-issued DOI.[^1] No peer-reviewed venue, code, or data is named. The review treats the paper as an unvalidated preprint proposal.

### 10.2 TurboQuant

TurboQuant’s record frames its contribution as near-optimal online vector quantization under mean-square and inner-product distortion, with empirical KV-cache results.[^3] It does not claim to be the optimal representation when arbitrary sequential side information and recomputation are free. Thus the reviewed paper is right that the problem classes differ, but wrong to present their rates as directly substitutable systems measurements.

### 10.3 PLT predecessor

The PLT record is another single-author preprint by the same author, submitted twelve days earlier, proposing probability-weighted tries for compression, policy, and caching.[^5] It is supporting author theory, not independent validation. The current paper inherits its unusual distance and prior-guided claims.

### 10.4 vLLM prefix caching

vLLM documentation hashes exact tokens and other prefix-affecting inputs to reuse blocks safely.[^4] This supports the paper’s exact-prefix baseline. It does not support approximate reuse of a different prompt’s KV, which would require an explicit distortion/quality contract.

### 10.5 Associated DEP record

No same-paper DEP was found. The live repository’s `DEP-A-20260713-CompressKV Semantic Heads` is a close conceptual alternative: it uses semantic retrieval signals to decide which KV heads to retain, whereas this paper proposes sequence coding of full KV. The relationship is methodological proximity, not shared authorship, implementation, or evidence.

The paper reports no measured systems results, so every practical performance statement above is assessed as a paper claim rather than an independently reproduced result. The time–space–distortion framing and the four hypotheses below are reviewer inference or proposals, not claims made by the author.[^6]

## 11. Research notes and critical considerations

### 11.1 Cache versus sufficient description

A cache is valuable because it trades memory for avoided computation. A source code proves only that information is representable. The correct object is a time–space–distortion tradeoff with an access model. This is the paper’s central missing abstraction.

### 11.2 Tokens are already the strongest side information

Inference systems retain token IDs or can reconstruct them cheaply. Therefore the theorem’s lowest-rate description is already present. Any new codec must show how small additional state reduces recomputation relative to token-only replay. Predictive deltas may do that, but no evidence is supplied.

### 11.3 Random access and dependency chains

Attention reads all prior KV every generated step. A sequential arithmetic code or predictor dependency can require decoding from a checkpoint. Practical designs need restart points, block-level indexes, bounded decode depth, and fused attention over compressed residuals. Those metadata and checkpoints increase rate.

### 11.4 Approximate prefix safety

Reusing centroid KV for a lexically different prompt changes the computation. A delta can restore exactness only if the full post-divergence delta is stored, which may approach a full cache. Omitting or coarsely quantizing it is lossy and must be evaluated for prompt instructions, security boundaries, and cross-tenant leakage.

### 11.5 Numerical and notational defects

The 80GB example, PLT direction, “early divergence” phrase, best-match maximize/minimize mismatch, and switch from inequality to entropy equality should be corrected before implementation. These are not cosmetic because they change system sizing and indexing behavior.

## 12. Limitations, implications, and applicability

### 12.1 Limitations

The paper’s stated limitations—no measured residuals and no kernel—are decisive. Additional limits include generic injectivity assumptions, unmeasured Lipschitz constants, no access model, no decode complexity, no experiments, distribution mismatch, incorrect sizing, and PLT inconsistencies. It also gives no code/data/funding/disclosure statement beyond references.

### 12.2 Scientific implications

The useful insight is to analyze activations conditionally and jointly. This could inspire blockwise learned predictors, entropy models, checkpointed replay, or residual codes. The exact-token entropy identity should be a starting point for time–space theory, not a headline compression ratio.

### 12.3 Deployment fit

No production use is justified yet. The ideas may suit research prototypes where long coherent documents, offline prefill, and large memory pressure permit extra predictor compute. Online high-concurrency decoding is especially sensitive to added candidate forward passes and serial decompression.

### 12.4 Governance

Cross-session deduplication must never cross tenant/security boundaries without cryptographic isolation and policy. Approximate sharing can make one user’s prompt state influence another. Compressed caches remain sensitive derived data and require deletion, retention, and provenance controls.

## 13. Independent falsifiable hypotheses

### Hypothesis 1: a cheap learned KV predictor beats vocabulary averaging

**Test:** train a small head from prior hidden state to next full-layer KV and compare storage/latency/quality with top-k candidate evaluation.  
**Prediction:** learned prediction captures most residual variance reduction at far lower compute.  
**Falsifier:** predictor error eliminates compression gains or destabilizes attention.

### Hypothesis 2: checkpointed token replay defines the real frontier

**Test:** store full/quantized KV every `B` tokens and reconstruct intervening blocks from tokens; sweep `B`.  
**Prediction:** replay offers a competitive memory reduction with predictable latency, clarifying whether residual coding adds value.  
**Falsifier:** even short replay blocks are slower than direct residual decoding at matched memory.

### Hypothesis 3: PLT score does not predict post-divergence KV similarity

**Test:** sample prompt pairs, measure the defined PLT value, lexical LCP, semantic similarity, and layerwise KV delta after divergence.  
**Prediction:** PLT adds little beyond lexical LCP and fails on paraphrases.  
**Falsifier:** it strongly predicts KV deltas after controlling for LCP length.

### Hypothesis 4: surprisal-guided bits improve direct residual quantization modestly, not by orders of magnitude

**Test:** quantize actual-minus-learned-predictor residuals with uniform versus surprisal allocation under equal average bits.  
**Prediction:** adaptive bits improve tail quality but realized ratios stay near single-digit multiples of direct quantization.  
**Falsifier:** stable hundredfold gains appear with negligible decode overhead.

## 14. Deployment and governance considerations

### 14.1 Required technical safeguards

- Define exact versus lossy reconstruction and attention/output distortion targets.
- Bound decoder operations and random-access latency per cache block.
- Insert restart checkpoints and verify corruption containment.
- Keep tenant-specific prefix namespaces and collision-resistant hashes.
- Measure predictor compute, memory traffic, throughput, and energy—not only stored bits.
- Fall back to exact or validated direct quantization on high-surprisal positions.

### 14.2 Required scientific safeguards

- Correct the formula/direction defects and publish executable proofs/tests.
- Verify injectivity/collisions on actual checkpoints.
- Report model/data distributions separately from model self-sampling.
- Calibrate KV MSE to attention logits and downstream tasks.
- Compare against token replay and optimized existing cache systems.

## 15. Replication and falsification agenda

### 15.1 Mathematical verification

Formalize the side-information and decoder-complexity model. Prove results with token-prefix conditioning without relying on unverified injectivity, then separately characterize when KV-prefix conditioning is equivalent. Correct PLT direction or redefine similarity. Recheck the rate–distortion dimension and the asymptotic condition.

### 15.2 Measurement-only phase

On Llama, Gemma, and Mistral, record next-token entropy, realized surprisal, exact candidate KV variance, layerwise residual norms, and PLT/KV-delta correlations. Export distributions by domain, position, layer, head, and context length. No compression claim should precede this phase.

### 15.3 Codec phase

Implement three predictors: top-k target evaluation, local linearization, and a learned head. Implement block checkpoints, synchronized bit allocation, and direct residual quantizers. Measure encoded bytes including metadata and restart state.

### 15.4 Systems evaluation

Compare fp16/bf16, KIVI/KVQuant/TurboQuant, exact prefix caching, token replay, and sequential variants at matched quality. Report prefill/decode p50/p95/p99, throughput, peak memory, bandwidth, energy, long-context accuracy, perplexity, and failure modes under concurrency.

### 15.5 Falsification thresholds

The practical thesis fails if reconstruction compute removes throughput gains, if PLT scores do not predict delta compressibility, or if residual codecs cannot beat direct quantization after metadata/checkpoints. The 914× practical claim fails unless an executable codec achieves it at matched downstream quality and bounded access latency.

## 16. Durable restatement

The paper’s durable observation is that a deterministic KV tensor sequence has no more symbolic uncertainty than the token sequence that generates it when the model and prefix are side information. This proves that independent vector coding is not the only information-theoretic formulation. It does not prove that a functioning KV cache can be compressed to token entropy without paying the transformer computation that caching was created to avoid. Probabilistic prefix sharing and predictive residuals are testable ideas, but the proposed PLT rule is internally inconsistent, predictor complexity is understated, headline ratios compare incompatible decoding models, and no experiment exists. The correct next step is a time–space–distortion theory plus an implementation—not deployment.
