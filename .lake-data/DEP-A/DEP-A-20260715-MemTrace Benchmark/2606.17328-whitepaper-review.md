# Holding the Fact Fixed While Memory Conditions Change

## A detailed review and independent re-conceptualization of “MemTrace: Probing What Final Accuracy Misses in Long-Term Memory”

**Source paper:** Xianxuan Long, Zhikai Chen, Shenglai Zeng, Shouren Wang, Kai Guo, and Jiliang Tang, arXiv:2606.17328v1.[^source]

**Artifact prepared:** 2026-07-15  
**Source reviewed:** complete 17-page arXiv v1 PDF and matching full-paper HTML, including benchmark appendices, twenty tables, and nine figures  
**Review scope:** benchmark reconstruction, aggregation and judge audit, system comparison, failure attribution, independent re-conceptualization, and replication agenda  
**Reproduction status:** benchmark paper and primary source records inspected; adapters, retrieval replay, oracle runs, and judge scoring were not independently executed

---

## Executive assessment

### The paper in one sentence

MemTrace changes long-term-memory evaluation from independent question rows to repeated probes of the same typed user fact across memory age, question type, and evidence condition, then distinguishes retrieval reach from failure to use reachable evidence.

### The deeper idea

The important contribution is a controlled evaluation design. If one row asks for a current preference, another asks for its prior state, and a third asks how it changed, pooled accuracy treats them as unrelated. MemTrace holds the **knowledge point** fixed and varies the surrounding conditions. That makes within-fact behavioral contrasts visible: recent versus aged, lookup versus trajectory, absent evidence versus a contradicted false premise.

### Bottom-line judgment

The paper makes a strong methodological case that final pooled accuracy is a lossy view of memory behavior. Its benchmark is large enough to expose system-specific patterns: 835 knowledge points from 20 users expand into 5,677 base probes, 15,422 question rows, and 200,453 scored answers across 13 configurations. The most consequential finding is that many errors occur after relevant evidence is reachable. The paper estimates that reached-but-unused failures are about ten times more common than unreachable-evidence failures.

The evidence is valuable but not universal. MemTrace derives its facts and diagnostic questions from HaluMem-Medium, uses LLM judges and answer generators, compares heterogeneous memory architectures with different backbones and interfaces, and uses a transparent dense-retrieval proxy for part of its reach/use attribution. The benchmark diagnoses fact-centered conversational memory; it does not by itself evaluate tool actions, multimodal episodic memory, permissions, or the truth of the underlying user statements.

### Principal strengths

- It makes the knowledge point—not the question row—the experimental unit.
- It separates retention, temporal use, missing-evidence refusal, and false-premise correction.
- It adds oracle and replay diagnostics to distinguish evidence availability from evidence use.

### Principal qualifications

1. Main scores depend on GPT-4o judging and a shared gpt-4o-mini answer generator; second-judge checks reduce but do not remove judge dependence.
2. The 13 systems are configurations across four paradigms, not perfectly matched implementations with identical backbones and interfaces.
3. “Evidence was reachable” is partly measured by a Text-emb-3-small replay proxy, so the 10× result is diagnostic rather than a universal causal decomposition.[^reach]

## 1. The problem the paper is actually solving

### 1.1 Aggregation erases behavior

Long-term-memory benchmarks commonly aggregate answer correctness over rows or episodes. This can rank systems but cannot show whether one underlying fact remains usable under changing conditions. A system can answer a current-state question correctly, miss the earlier state, and fail to describe the update trajectory. Another can refuse questions about absent facts while accepting a user’s false premise about a remembered fact. The same pooled score can hide both profiles.

### 1.2 Why the unit of analysis matters

MemTrace defines a knowledge point as one typed fact about a user. Repeated probes are attached to that identity. This creates a panel-like design: the object remains fixed while memory window, question form, and evidence condition vary. Comparisons are then interpretable as behavior changes for the same fact rather than differences between unrelated questions.

### 1.3 The proposed gap

The benchmark targets three axes:

- **Memory age:** how many sessions separate a fact’s source from the evaluation checkpoint.
- **Question type:** current state, historical state, or trajectory of change.
- **Evidence condition:** supported, missing, or contradicted by a false premise.

The authors further ask where failure originates. Is the gold evidence unreachable, or did the system retrieve it and still fail to use it? This moves the analysis from “which system scored higher?” to “which memory behavior broke?”

## 2. Formal and technical reconstruction

### 2.1 Data source and knowledge-point construction

MemTrace uses HaluMem-Medium as source data because it contains multi-session user histories, session-anchored facts, and distractors suitable for missing/conflict probes.[^halumem] The construction converts memory points into substantive knowledge points—static, dynamic, and preference facts—and distractor knowledge points for conflict and boundary conditions.

The final distribution is 348 static facts, 213 dynamic facts, 74 preferences, 100 conflict distractors, and 100 boundary distractors: 835 total. Of these, 635 are substantive user facts and 200 are controlled distractors. Each of 20 users contributes 35–48 knowledge points, mean 41.8.

### 2.2 Memory windows and age

Each history is evaluated at eight chronological checkpoints (W_1,\ldots,W_8). A window exposes only sessions before its evaluation boundary, preventing future leakage. A knowledge point is probed only after its initial appearance. If its source is session (t_{\mathrm{source}}) and the window ends at (t_{\mathrm{eval}}), memory age is (t_{\mathrm{eval}}-t_{\mathrm{source}}).

The checkpoints span early to saturated histories, with extra placement around dynamic updates. They are cumulative prefixes, not disjoint blocks. This design traces degradation as more interactions accumulate.

### 2.3 Question types

Substantive facts receive current, historical, and trajectory questions. Current questions ask the latest state. Historical questions ask an earlier state. Trajectory questions ask how the fact changed. Distractor knowledge points use current and historical forms appropriate to missing/conflict evaluation.

Trajectory is qualitatively different from retrieving one answer string. It requires finding multiple states, ordering them, and expressing the transition. This distinction drives several of the paper’s strongest findings.

### 2.4 Evidence conditions

Standard probes have relevant evidence. Boundary probes ask about facts absent from the history and reward appropriate abstention. Conflict probes assert a premise that contradicts remembered evidence and require correction. The distinction matters: refusing unsupported content is not the same capability as using memory to oppose a false premise.

### 2.5 Response scoring

Each response receives a tuple ((g,v,r)). (g) is binary **Gist accuracy**, intended to capture semantic correctness. (v\in[0,1]) is **Verbatim completeness**, based on canonical answer-token coverage. (r) is a response-type label supporting correct, abstention, and hallucination diagnostics. GPT-4o is the primary judge; Gemini-3-Flash re-scores a stratified 200-probe sample under the same rubric.

### 2.6 Diagnostic views

Memory maintenance uses full (W_1\ldots W_8) traces plus **Fresh** accuracy over (W_1,W_2), **Saturated** accuracy over (W_7,W_8), and (\Delta\mathrm{Forget}=\mathrm{Fresh}-\mathrm{Saturated}). The paper correctly warns that a small gap can mean stable excellence or uniformly poor accuracy.

Evidence-condition behavior reports conflict correction, boundary abstention, and hallucination separately. Failure attribution defines retrieval reach (R) and solve outcome (U). Cases with (R=1,U=0) are “reached but unsolved.” Oracle runs supply gold evidence directly to test whether an answer generator can recover.

## 3. Architecture and information flow

MemTrace is a benchmark pipeline rather than a new memory architecture. Source sessions are converted to typed knowledge points, expanded into base probes, rendered into question rows, passed through a memory-system adapter, answered by a shared generator where applicable, judged, and aggregated into diagnostic views.

The main comparison spans:

- long-context models that read visible history;
- BM25, dense, and graph RAG systems;
- external-memory stores such as Mem0, SimpleMem, REMem, and AMem;
- agentic-memory systems MIRIX and Mem-T.

For non-long-context systems, memory state is built only from the visible prefix. Retrieved or stored evidence is passed to a shared gpt-4o-mini answer generator, which helps isolate memory-interface differences while introducing a common generator dependency. Paper-native sensitivity rows are shown separately rather than mixed into the main system count.

### Guarantee boundary

The benchmark guarantees neither that a system’s stored memory is correct nor that LLM judge labels are ground truth. Quality-control steps reduce construction defects, and windowing constrains future leakage, but benchmark validity still depends on source labels, prompt generation, adapter implementation, and judge reliability.

## 4. Experimental design reconstructed

### 4.1 Scale and workload

The 835 knowledge points become 5,677 base probes and 15,422 question rows. Across 13 completed configurations, the paper reports 200,453 scored system-answer rows. The repeated-measures structure is the important feature: each fact contributes observations under multiple controlled conditions.

### 4.2 System configurations

Long-context configurations include Qwen3.5-35B, Gemini-3-Flash, and GPT-5-nano. RAG uses BM25, Text-emb-3-small, Qwen3-Emb, and HippoRAG-v2. External memory includes Mem0, SimpleMem, REMem, and AMem. Agentic memory includes MIRIX and Mem-T. Appendix B records configuration, backbone, conflict path, and temporal path.

The matrix is transparent that “yes,” “partial,” and “no” are descriptive implementation labels, not standalone capability claims. Cross-system results should still be interpreted cautiously because extraction, storage, retrieval interface, and model size vary.

### 4.3 Aggregation

Fresh and Saturated Gist are reported per question type and overall. The appendix adds bootstrap confidence intervals and rank-correlation analysis. User-level macro aggregation and mean-over-question-types conventions appear in sensitivity tables. A reproduction must preserve these denominators; averaging all rows would overweight knowledge points that generate more probes.

### 4.4 Judge reliability

The paper cross-checks a stratified 200-probe sample with Gemini-3-Flash and reports agreement, Cohen’s kappa, confidence intervals, and disagreement breakdown. This is good practice. It remains a limited sample under the same rubric, so shared prompt ambiguities can affect both judges.

### 4.5 Failure-origin design

Three checks support attribution. First, gold evidence is supplied on probes all systems miss, testing whether they are answerable. Second, a transparent dense retriever replays evidence reach. Third, oracle evidence is supplied on reached-but-unsolved cases. Large oracle recovery after (R=1,U=0) supports an evidence-use bottleneck.

The replay is not the production retriever for every system. It is a common proxy. The paper appropriately labels it transparent rather than claiming to observe every system’s internal reach.

## 5. Results: what is reported and what it means

### 5.1 Memory maintenance

No single system dominates every question type. HippoRAG-v2 has the strongest overall Saturated Gist at 36.5% and leads saturated current and historical questions at 45.4% and 50.9%. Mem-T leads saturated trajectory at 19.8%. The low absolute values are as important as the rankings: long-term temporal use remains difficult.

Long-context models show striking trajectory degradation. Qwen3.5-35B falls from Fresh trajectory 49.0% to Saturated 6.7%; GPT-5-nano falls from 38.4% to 6.5%. HippoRAG-v2’s saturated trajectory is 13.4% despite strong current/historical results. These patterns support the paper’s thesis that retrieving one state and organizing an update trace are different capabilities.

### 5.2 Missing versus conflicting evidence

Mem0, AMem, and REMem have boundary abstention rates of 99.3%, 97.4%, and 94.0%, showing that they usually refuse questions about unmentioned facts. Their conflict Gist scores are only 14.6%, 20.1%, and 35.1%. Conflict hallucination is relatively low—2.7%, 1.1%, and 4.9%—because many failures are abstentions.

This exposes a crucial distinction. A system may look safe when evidence is missing but treat contradiction as if evidence were absent. Correcting a false premise requires the memory-to-answer interface to recognize contradiction and use evidence assertively.

### 5.3 Reach versus use

Across the diagnostic sample, failures attributed to reached-but-unused evidence occur roughly ten times as often as unreachable evidence. Oracle runs on (R=1,U=0) cases often recover when gold memory is placed directly in the answer context. This supports the conclusion that more storage or broader retrieval alone will not solve the dominant remaining errors.

The result should be read as a benchmark diagnostic. Reach is measured with a proxy and “use” includes answer-generation, evidence formatting, contradiction handling, and multi-state reasoning. It does not isolate one internal neural mechanism.

### 5.4 Confidence and sensitivity

Appendix tables provide bootstrap intervals for saturated Gist and conflict/boundary rates, rank-correlation distributions, prompt-control checks, backbone swaps, oracle sensitivity, and retrieval-interface audits. These strengthen the main pattern while also demonstrating that conflict resolution is sensitive to the answer backbone and how evidence is exposed.

One particularly informative intervention surfaces Mem0 retrieval similarity scores. It reduces backbone coupling on conflict Gist but raises Gemini conflict hallucination from 7.1% to 12.1% and lowers boundary abstention from 88.7% to 68.8% on the audit subset. Better evidence use can trade against conservative refusal; interface changes need multi-metric evaluation.

## 6. Ablations and causal evidence

MemTrace’s “ablations” are largely benchmark sensitivity and diagnostic interventions rather than ablations of one model. Prompt-control variants test whether horizon and question wording explain retention patterns. Backbone swaps test whether conflict and oracle findings depend on the answer model. Oracle routing uses benchmark labels as non-deployable signals to ask whether diagnostic axes can guide targeted interventions.

The evidence-interface audit is mechanistically useful. Systems return different forms of retrieved evidence—raw chunks, summaries, scored items, or structured memory. Changing visibility of similarity scores changes conflict behavior and safety trade-offs. This supports the paper’s claim that the bottleneck often lies between reachable evidence and answer production.

The design does not establish one causal treatment effect across all systems. Each diagnostic changes a specific interface or condition, and heterogeneity is part of the result.

## 8. External context

The official arXiv record identifies MemTrace as a June 2026 v1 preprint.[^source] HaluMem’s primary paper defines operation-level extraction, updating, and QA hallucination tasks and supplies the multi-session source data MemTrace transforms.[^halumem] MemoryAgentBench independently decomposes memory into accurate retrieval, test-time learning, long-range understanding, and selective forgetting, reinforcing the broader move away from one pooled score.[^mab]

LoCoMo provides an earlier long-term conversational benchmark with human-edited temporal event graphs, but its conventional row-level QA aggregation does not provide MemTrace’s within-knowledge-point panel.[^locomo] MemTrace’s name should not be confused with the separate 2026 paper “MemTrace: Tracing and Attributing Errors in Large Language Model Memory Systems,” which studies executable memory-evolution graphs and has a different arXiv identity.

No official code URL was verified in the inspected MemTrace paper metadata. The appendix is detailed, but absence of a verified repository blocks direct audit of adapters, prompt rendering, aggregation code, and the exact replay/oracle pipeline.

## 9. Re-conceptualization: a longitudinal panel for memory facts

MemTrace is best understood as a **longitudinal panel study over knowledge points**. The fact is the panel entity. Windows are time observations. Question forms and evidence conditions are controlled treatments. System responses are outcomes. This view explains why row-level accuracy is inadequate: it discards entity identity and condition structure.

The analogy has limits. Conversational facts are constructed labels rather than naturally sampled people, and LLM-generated questions/judgments introduce measurement error. Still, the panel framing yields actionable analysis: within-fact forgetting, change-tracking, contradiction response, and failure-origin decomposition.

A transferable principle follows: evaluations of persistent agent state should preserve the identity of what is remembered and vary the operational condition around it. This applies to user preferences, plans, credentials, evolving documents, tool outcomes, and learned procedures.

## 10. Research notes and critical considerations

### 10.1 Measurement dependence

Gist is an LLM-judge label, Verbatim rewards canonical token presence, and response type encodes abstention/hallucination categories. A valid paraphrase can challenge token completeness; an ambiguous trajectory can challenge binary Gist. Human audits should target disagreements by question type and evidence condition, not just a pooled sample.

### 10.2 Correlated observations

Repeated probes from one knowledge point and user are correlated. Bootstrap procedures should resample at the user or knowledge-point level rather than treating 200,453 rows as independent. The paper supplies user-level and bootstrap views, but every reproduced interval must preserve clustering.

### 10.3 Source-data transformation

MemTrace inherits HaluMem-Medium histories and performs manual fixes. Table 7 documents pre/post quality control, which is good. Remaining source artifacts or transformations could affect difficulty. A release should preserve immutable knowledge-point provenance and every edit.

### 10.4 Interface versus memory

The reach/use result shows that what a benchmark calls “memory performance” includes evidence serialization, prompt instructions, answer backbone, and conflict policy. This is not a flaw; it is a reminder to report the whole system path.

### 10.5 Distribution scope

Twenty users enable repeated evaluation but remain a small population. Facts are text-based and user-centric. Deployment claims should not extend to multimodal experience, enterprise knowledge, tool-state memory, or culturally diverse personalization without new evidence.

---

## 11. Potential implications

For research, benchmark leaderboards should publish per-fact trajectories, not only means. For system design, memory APIs should expose retrieved evidence and support diagnostics for state age, question intent, missing support, and contradiction. For operations, monitoring can distinguish “not stored,” “not retrieved,” “retrieved but ignored,” and “used incorrectly.”

For governance, boundary abstention and conflict correction should be separate acceptance criteria. A system that refuses unsupported questions may still comply with a false user premise despite contrary memory. Logging needs to preserve the evidence presented and the response policy applied.

## 12. New hypotheses derived from the paper

These are reviewer inferences, not established findings.

### Hypothesis 1: explicit state-delta representations improve trajectory questions

**Proposition:** Memory stores that represent updates as linked deltas will outperform flat retrieval on saturated trajectory probes.  
**Predicted observation:** Current/historical accuracy changes little while trajectory Gist rises.  
**Falsifier:** Delta links provide no gain under matched evidence budget.  
**Minimum test:** Add predecessor/successor edges to a dense-RAG baseline and evaluate the same knowledge points.

### Hypothesis 2: contradiction routing needs a distinct action policy

**Proposition:** A dedicated “correct false premise with evidence” route will improve conflict Gist without sacrificing boundary abstention.  
**Predicted observation:** Conflict correctness rises while boundary hallucination stays stable.  
**Falsifier:** Improvements require accepting more unsupported claims.  
**Minimum test:** Train or prompt a three-way support/missing/conflict router under held-out facts.

### Hypothesis 3: reach/use attribution changes with evidence interface

**Proposition:** Structured evidence carrying source, time, and contradiction labels will reduce (R=1,U=0) more than increasing retrieval K.  
**Predicted observation:** Oracle gap narrows under fixed recall.  
**Falsifier:** Larger K is equally effective at matched context length.  
**Minimum test:** Cross evidence serialization and K in a factorial experiment.

## 13. Deployment and governance considerations

MemTrace is an evaluation methodology, not a deployment component. Its operational use would require an immutable mapping from source events to knowledge-point IDs, privacy controls, retention/deletion propagation, reproducible probes, judge calibration, and a policy for disputed or corrected user facts.

Safeguards should include user-level isolation, source provenance, time-aware updates, conflict logging, evidence-visible answers, abstention diagnostics, judge disagreement sampling, and clustered uncertainty. Passing MemTrace does not constitute security certification or permission to retain sensitive user information.

## 14. Replication and falsification agenda

1. Release the exact knowledge-point ledger, source-session anchors, manual corrections, probes, adapters, prompts, and aggregation code.
2. Recompute all counts and verify no future-session leakage at each window.
3. Re-score a clustered human sample stratified by question type, evidence condition, system, and judge disagreement.
4. Reproduce Fresh, Saturated, and trajectory tables using user/knowledge-point bootstrap intervals.
5. Run native-retriever reach logging in addition to the Text-emb-3-small replay proxy.
6. Audit (R=1,U=0) cases by evidence serialization and answer backbone.
7. Add matched-backbone/matched-budget variants for representative long-context, RAG, external, and agentic systems.
8. Test multilingual users, longer histories, tool-state facts, permission changes, multimodal events, and deliberate contradictions.
9. Measure benchmark contamination and soft overlap, especially where released memory policies trained on LoCoMo-lineage data.
10. Report privacy and compute costs for building and scoring the full panel.

The headline conclusion would weaken if native-retriever logs showed that most “reached” evidence was not actually available to production generators, or if human labels substantially disagreed with the conflict and trajectory judgments.

---

## 15. Durable restatement

> MemTrace evaluates long-term memory by repeatedly probing the same user fact instead of averaging unrelated question rows. It shows that recent lookup, aged retention, trajectory reasoning, missing-evidence refusal, and false-premise correction are distinct behaviors. Its diagnostic samples suggest that reachable evidence is often not used effectively, making the memory-to-answer interface a central bottleneck.

The lasting contribution is the evaluation unit. If memory persists over time, the benchmark must preserve the identity and history of what is remembered.
