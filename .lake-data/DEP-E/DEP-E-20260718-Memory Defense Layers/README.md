## Insights and Relevance

The deposit shifts the earlier memory-forensics thread from trajectory classification toward authority-aware system design. Its strongest reusable relationship is that persistent state needs provenance, scope, and action authorization: a model can refuse execution while still leaving dormant adversarial state for another model, agent, or future version. The paper also demonstrates why agent evaluation must record effective tool schemas, retrieval contents, runtime state, and pre-activation equivalence. These findings support provenance-bearing memory gateways and defense-attribution harnesses more directly than a universal trace blocker.
