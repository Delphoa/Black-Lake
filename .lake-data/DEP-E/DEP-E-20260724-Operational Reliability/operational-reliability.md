## Executive Summary

The selected DEP contains ten July 2026 works that appear diverse—agent memory, context garbage collection, transformer architecture, open-world tools, safety pragmatics, skill supply chains, coding benchmarks, rare-disease diagnosis, geochemical simulation, and distributed quantum computing. Read together, they describe one operational problem: a system is not reliable merely because its model or component scores well. Reliability depends on what state is retained, which instructions and dependencies have authority, how tools and environments shift, whether evaluation labels preserve the cause of failure, and whether aggregate improvements hide lost items or uncontrolled experimental assumptions.

Three papers address state directly. AutoMem reports that file-system memory operations can be treated as an observable, trainable skill; its Qwen2.5-32B agent rises from progression rates of 25.00/7.50/0.42 to 51.36/30.00/1.85 across Crafter, MiniHack, and NetHack after scaffold optimization and memory-specialist training (E1). Self-GC treats user turns, tool outputs, and skill state as indexed lifecycle objects; on its 33-session Hard Set it prunes 43.95% of prefix tokens with an 84.85% judge-rated no-impact rate, compared with 54.55%-69.70% no-impact for more aggressive heuristics (E2). State-Prediction Separation moves the argument into the Transformer: controlled pretraining at 53M-1.678B parameters reports better validation loss and roughly 2-3 percentage-point average downstream gains when persistent state preparation is separated from immediate prediction, though at roughly double per-step training compute (E3).

Four works show why reliability must preserve failure type and provenance. OpenAgent reports that both supervised and reinforcement-trained tool agents degrade across query, action, observation, rule, and domain shifts; perturbation-augmented training helps but does not remove all late-stage degradation (E4). Adversarial Pragmatics separates task success, source sensitivity, policy compliance, safety risk, refusal outcome, and failure attribution; its small pilot also shows that a rubric-aided LLM judge missed the minority safety-relevant classes even under favorable conditions (E5). The skill-supply-chain study reports 0.95 overall F1 for its analyzer and finds that many security signals are visible only through transitive skill, package, or service dependencies (E6). The coding-benchmark audit finds that only 39/102 GSO, 11/140 SWE-Perf, and 411/498 SWE-fficiency reference patches satisfy their original rules in every cross-machine replay, while ranking changes depend materially on penalty design (E7).

The three high-stakes or scientific studies reinforce the same discipline. RareDxR1 reports 52.29% Top-1 and 69.27% Top-10 on RareArena-Test after supervised and curriculum-reinforcement stages, but its promised code and data remain unavailable at the inspected repository pin, so the result is not independently reproducible and is not clinical validation (E8). PHREEQC-MCQ-200 shows that simulator tools produce large net gains for capable agents while still losing 10-32 items that no-tool versions answered correctly; output interfaces help top-tier models and hurt mid-tier ones (E9). The quantum paper reports more than 20% improvement under layered mitigation, yet the experiment emulates fiber/transduction noise on a single IBM backend and shows purification alone approaching the full stack at higher noise (E10).

Reviewer interpretation: these works support an operational reliability stack with four controls: versioned state, explicit authority and dependency graphs, diagnostic evaluation with per-item accounting, and domain-specific evidence gates. They do not establish a universal architecture, clinical readiness, secure deployment, or scalable distributed quantum advantage. No result was independently reproduced in this pass.

## Detailed Summary

### 1. State as an engineered substrate

AutoMem distinguishes the structure supporting memory from the model's proficiency in using it. Its inner agent promotes read, write, append, search, and create operations into the same observable action space as world actions. A first outer loop gives complete trajectories, memory directories, and scaffold code to a stronger meta-LLM, which revises prompts, code, schemas, and action vocabulary only when fixed-seed progression improves. A second outer loop selects the base model's own useful memory decisions as supervised examples and trains a LoRA memory specialist while leaving the gameplay model unchanged. The released repository exposes evolved per-game scaffolds and the training-engine components. Reported progression rises from the memory-file baseline at 25.00%, 7.50%, and 0.42% to 47.27%, 27.50%, and 1.57% after scaffold optimization, then 51.36%, 30.00%, and 1.85% after memory training for Crafter, MiniHack, and NetHack respectively. The comparison is compelling within the BALROG game harness, but each game receives a separate scaffold and specialist, the fixed evaluation uses ten seeds with environment-specific episode counts, and the outer loops depend on proprietary meta-LLMs.

Self-GC models accumulated context as typed, indexed objects rather than a suffix of tokens. A side-channel planner proposes fold, mask, and prune operations. The harness rehearses the proposed plan, rejects invalid or latest-user-turn edits, materializes the projected context, estimates savings, writes recoverable sidecars, and commits only at cache-aware boundaries. This separation is important: the model proposes semantic cleanup, while deterministic code owns validity and recoverability. The 332-session Production Suite and 33-session Hard Set are derived from 15,141 trace rows after filtering. On the Hard Set, Self-GC prunes less than heuristics (43.95% versus 61.90%-69.87%) but reaches higher no-impact (84.85% versus 54.55%-69.70%). Across three planner backbones on the Production Suite, no-impact is 91.27%-94.58% while pruning 31.04%-33.98%. The production account split reports 10%-15% daytime average input-token reductions and peaks near 20%, but it is monitoring evidence, not a randomized quality experiment, and the private traces prevent open replay.

The State-Prediction Separation paper asks whether the same hidden computation should both emit the next-token prediction and populate state that future tokens read. Its SPS variant creates input and prediction streams: persistent state is prepared from the input stream while next-token emission uses an ephemeral prediction stream. The study controls parameter count and compares Standard, SPS, 2x Memory, Delayed State, and Reverse SPS across 53M, 131M, 379M, 831M, and 1.678B parameters trained on FineWeb-Edu. It evaluates held-out loss, four out-of-distribution corpora, and five zero-shot tasks. Ablations indicate that simply doubling persistent memory or delaying state does not match SPS, and gradient analysis supports a different future-loss routing mechanism. The public release includes an MIT license, `uv.lock`, training configs, evaluation tasks, and figure scripts. The evidence remains bounded by a single pretraining corpus, a single main-run seed, an under-trained largest model, and roughly doubled training compute per input position.

### 2. Open-world control, safety labels, and dependency provenance

OpenAgent formalizes tool-use shifts along query, action, observation, and domain dimensions, organized into Perception, Interaction, Reasoning, and Internalization tiers. The controlled sandbox changes names and descriptions, inserts distractors and errors, redirects tools, modifies execution graphs and rules, removes required information, and transfers task structure to a medical-tool domain. Results show degradation after both SFT and RL, including the striking loss of active refusal after clean static training. Perturbation-Augmented Fine-Tuning combines environment-feedback perturbation, solvability-boundary perturbation, and symbolic-representation perturbation. At an early SFT stage it changes the hardest Tier 1-3 deltas from -67.7/-48.2/-39.9 to improvements of +28.6/+26.5/+22.7 and restores a 99.3 refusal rate in the reported condition. At later stages, refusal remains high but several Tier 1-3 deltas remain negative, showing that coverage-based augmentation is targeted mitigation rather than a universal solution.

Adversarial Pragmatics addresses an evaluation-layer ambiguity: identical strings can be instructions, quotations, tool data, policy examples, or transcript evidence. The current v2 defines eight families covering embedded commands, use/mention, authority, scope, reference, indirect speech acts, policy-boundary ambiguity, and transcript interpretation. Its 18 safe seed items use harmless color tokens and dummy secrets. The 54-row pilot over three local models produces 36 full successes, 46 policy-compliant outputs, and 12/24 eligible strict-pair passes after excluding one defective contrast from strict scoring. The LLM-judge audit is more informative than its headline agreement: safety-risk agreement is 72.2% against a 79.6% majority-class base rate, with zero recall on 11 risk-labeled rows; it also misses all 11 partial task successes. The judge graded its own outputs and saw expected-behavior fields, so this is an upper-bound warning, not a deployable judge result. The repository retains sanitized summaries and explicitly excludes raw local model bundles.

The Agent Skill Supply Chain study treats a skill as a mixed document containing metadata, natural-language instructions, and executable scripts. SkillDepAnalyzer extracts and resolves typed dependencies across skills, packages, and external services, emits a SkillBOM, and preserves low-confidence candidates as annotations. On the 500-root single-layer SKILL-DEP benchmark it reports 0.95 overall F1, versus 0.52 for a DeepSeek semantic extractor and 0.18-0.25 overall F1 for the package-oriented SBOM tools shown. Its 1.43 million-skill analysis reports non-unique names for 58.73% of skills, hidden transitive package inventory rates of 71.87% for npm and 73.33% for PyPI, and dependency clusters among 30.41% of dependency-bearing roots. Security findings are explicitly audit signals rather than confirmed vulnerabilities: 60%-78% of affected skill roots, 98.01% of axios-root exposure, and 93.10% of vulnerable-MCP-service hits arise only through dependency paths. Construct and external validity remain limited by SDA's dual role as instrument and object, SkillsMP/GitHub coverage, rapidly changing registries, and exclusion of private enterprise skills.

### 3. Evaluation must preserve machine, item, and protocol effects

The coding-benchmark audit replays 740 official optimization reference patches on four 64-vCPU, 256-GB Google Cloud profiles spanning Intel and AMD generations. Its strict rule accepts a task only when the original benchmark validity condition holds in all 12 machine-round replays. That leaves 39/102 GSO tasks, 11/140 SWE-Perf tasks, and 411/498 SWE-fficiency tasks. The score audit then shows that leaderboard order is not simply agent capability: among eight shared submissions, GSO and SWE-fficiency disagree on 9/28 pairwise comparisons; SWE-fficiency's worst ten tasks carry 58.5%-82.8% of harmonic-mean score weight; and capping one-task penalty moves six of eight ranks and flips 8/28 pairwise orders. The public repository provides compact task and replay tables but intentionally omits raw replay logs and the internal pipeline, which enables count verification but not full execution reproduction.

PHREEQC-MCQ-200 makes a parallel point for scientific tools. Each of 200 four-option questions comes from one of 21 expert-reviewed scenario stems and requires an agent to write a PHREEQC input, execute version 3.8.6 against a frozen author-supplied database, inspect structured output, and commit a single answer letter within 24 steps. Direct model baselines range from 27.0%-42.0%. Capable tool agents reach 54.0%-84.0%, adding 15.0-41.5 percentage points over the better no-tool baseline. Yet tool use is never purely additive: the reported agents lose 10-32 previously correct items while gaining more, for retention rates of 56.4%-86.5%. A table-of-contents output interface saves 11%-57% input tokens and matches or improves accuracy by 0.5-2.5 points for top-tier models, but costs GPT-5.1 and GPT-5.2 7.5-9.5 points. A below-mid model regresses below its no-tool baseline. The study therefore treats output access, item retention, rerun stability, step budget, and failure stage as separate measurements.

### 4. High-stakes transfer requires stronger evidence gates

RareDxR1 bypasses a structured phenotype or retrieval pipeline by internalizing rare-disease knowledge into model parameters and generating reasoning trajectories through Reflection-Enhanced Reasoning Sampling. It applies supervised training followed by dual-level curriculum reinforcement learning that varies task difficulty and case difficulty while interleaving tiers to reduce forgetting. Evaluation uses RareArena-Test (1,025 cases, 637 diseases), MIMIC-IV-Rare (590 cases, 239 diseases) as an out-of-distribution set, RareBench-Public, and DiagnosisArena. On the reported RareArena ablation, the DeepSeek-Distill base moves from 25.76/35.12/42.15 Top-1/3/10 to 52.29/63.61/69.27 with supervised plus curriculum RL; zero-shot Top-10 recall is reported as 50.00% versus 28.85% for DeepSeek-R1-671B. Those are research results, not clinical validation. The inspected official repository contains only a placeholder README, so the promised code, dataset, model, preprocessing, privacy handling, calibration, and audit surface remain unavailable.

The quantum-data-center paper evaluates a layered mitigation stack on the 156-qubit IBM `ibm_kingston` Heron r2 backend. Instead of physically connecting two QPUs, it emulates transduction and G.652-D fiber effects through a Markovian collisional model, mapping ten interaction steps to ten-meter segments. The experiment uses 10,000 shots and combines BBPSSW-style entanglement purification, Pauli twirling, and readout-matrix inversion around a cat-state remote CNOT protocol. In Figure 3, baseline Bell fidelity falls from roughly 0.97 to 0.42 at step ten; purification retains about 0.63. Remote-gate success at step ten reaches roughly 0.58-0.62 with purification and twirling, while the complete stack is usually best. The paper also reports that purification alone approaches the complete stack at greater noise, implying communication-resource quality dominates local mitigation there. The work is experimental on real hardware but not on physical fiber-linked QPUs, and it provides no public code, calibration snapshot, raw counts, uncertainty bars, or independent-device replication.

## Methodology

- `Research objective`: Preserve and evaluate the selected DEP's ten-source bundle as a schema-complete DEP research artifact, identifying shared operational-reliability mechanisms, evidence strength, implementation implications, and validation gaps.
- `Sources inspected`: Both selected DEP files; canonical arXiv records for all ten works; complete arXiv HTML for nine papers; the complete four-page PDF and every rendered page for the quantum paper; and official public repositories for AutoMem, SPS, OpenAgent, adversarial pragmatics, coding-benchmark reliability, and RareDxR1.
- `Discovery strategy`: Repository inspection established the source inventory. Canonical arXiv records resolved current versions and full text. Paper links and explicit availability statements identified official repositories. Current repository HEADs were resolved and recorded as commit pins.
- `Inclusion criteria`: All ten primary works listed in the source DEP were included. Official implementations or data repositories were included only when the paper or canonical record identified them.
- `Exclusion criteria`: Secondary summaries, search-result snippets, unverified code mirrors, unrelated repositories, and merely recommended papers were excluded as evidence. No private trace, clinical record, model weight, source dataset, benchmark payload, or experiment output was collected.
- `Analytical approach`: Mixed conceptual, empirical, comparative, implementation, safety/ethics, product-research, and replication analysis.
- `Extraction process`: Methods, experimental settings, tables, figures, results, limitations, data/code statements, and repository inventories were read directly. The quantum PDF was text-extracted and visually inspected page by page because HTML conversion failed.
- `Version control`: The selected source DEP is pinned to `0f6545b`; current arXiv versions are explicit; six official repositories are pinned to the commit SHAs in Source Metadata.
- `Claim selection`: Central mechanisms, quantitative results, cross-source tensions, release status, and boundary conditions were prioritized. Aggregate metrics were paired with denominator, protocol, scale, or item-retention context where available.
- `Evidence handling`: Each central claim maps to an evidence-ledger item. Source claims, reviewer interpretation, and cross-source inference are labeled separately.

- `Safety handling`: Safety examples remain harmless and defensive; clinical content is framed as decision-support research rather than medical advice; no harmful agent payload, private patient record, or operational quantum-control procedure is reproduced.

## Observations

- `Observed pattern - state needs ownership`: AutoMem assigns memory decisions to the agent and optimization loops; Self-GC assigns commit authority to a deterministic harness; SPS assigns persistent-state preparation to a distinct computation stream. All three reject passive context accumulation.
- `Observed pattern - reliability is typed`: OpenAgent's four tiers, adversarial pragmatics' label families, the SkillBOM dependency channels, coding-benchmark replay validity, and PHREEQC failure stages all replace a single success bit with a structured diagnosis.
- `Observed pattern - aggregate gains trade items`: AutoMem's overall progression, PHREEQC's tool gains, leaderboard harmonic means, RareDxR1's recall, and quantum fidelity summaries can hide seed, item, task, class, or noise-regime losses. The papers with the strongest diagnostic value expose those substructures.
- `Technical implication`: A reliable agent runtime needs separate schemas for state objects, instruction authority, dependency provenance, tool outcomes, evaluator judgments, and final artifacts. Combining them in one transcript or score makes later attribution difficult.
- `Technical implication`: The harness should own invariants—last-turn retention, recoverability, allowed tool graph, dependency pins, output schema, and stop conditions—even when a model proposes semantic changes.
- `Contradiction or tension`: Automation reduces review cost but moves trust into meta-LLMs and model judges. AutoMem relies on meta-LLM trajectory review, Self-GC on a judge for no-impact, and adversarial pragmatics shows that a favorable LLM judge can miss safety-relevant minority labels.
- `Contradiction or tension`: More computation or more tools are not monotonic improvements. SPS pays roughly double per-step compute; PHREEQC tools lose previously correct items; PAFT can dilute clean supervision; quantum layers add limited value beyond purification at higher modeled noise.
- `Open question`: Can one common evidence contract represent memory decisions, tool traces, benchmark validity, dependency lineage, and domain-specific measurements without flattening their distinct semantics?
- `Reviewer hypothesis`: A practical reliability score should be a vector of invariants and retained-item rates, not a scalar. Any aggregate should be computed only after the deployment-specific utility and penalty policy are explicit.

## Considerations

- `Evaluation validity`: Report denominators, confidence intervals, item-retention matrices, failure stages, and machine or endpoint conditions. A single average or rank should not be treated as system capability.
- `Judge governance`: Validate judges on minority classes, use independent adjudicators, blind expected labels where appropriate, and preserve abstention or ambiguity rather than forcing consensus.
- `State privacy`: Context objects and memory files can contain user secrets, private traces, or clinical details. Retention, sidecars, and recovery paths require access control, encryption, minimization, and deletion policy.
- `Dependency security`: A root skill review is insufficient when instructions can import other skills, packages, or services. Resolve the full graph, pin identities and versions, record evidence, and block uncertain high-authority dependencies.
- `Tool robustness`: Test name changes, schema shifts, missing information, altered execution graphs, output-size extremes, permission failures, and honest refusal conditions.
- `Medical governance`: Rare-disease reasoning systems require prospective clinical evaluation, privacy review, subgroup analysis, calibration, clinician oversight, provenance, and a safe abstention pathway. Benchmark accuracy is not a care recommendation.
- `Scientific reproducibility`: Freeze simulator versions, databases, tool interfaces, random seeds, step budgets, and expected outputs. Record items lost as well as gained.
- `Quantum interpretation`: Emulated communication noise on one monolithic device should not be described as a deployed fiber-linked quantum data center. Calibration drift, post-selection yield, uncertainty, and physical transduction loss must be measured independently.
- `Maintenance`: Public repositories with one or a few commits, absent releases, missing locks, or placeholder content are not frozen reproduction packages. Use commit pins and preserve release-status notes.
- `Cost`: Reliability controls introduce planner calls, judges, graph resolution, replay hardware, additional training streams, or mitigation overhead. Cost claims must include the entire control stack.

## Strengths

- The bundle spans mechanism, runtime, safety, supply chain, evaluation, and domain transfer, enabling a fuller reliability model than any single paper.
- AutoMem, SPS, OpenAgent, the coding-benchmark audit, PHREEQC-MCQ-200, and the quantum study expose concrete experimental configurations and quantitative results.
- Self-GC, adversarial pragmatics, and PHREEQC foreground diagnostic structure rather than only aggregate performance.
- The skill-supply-chain study connects natural-language agent artifacts to established SBOM ideas while preserving skill, package, and service distinctions.
- Several sources provide official repositories; the SPS release includes a dependency lock, and OpenAgent includes model-free tests.
- The adversarial-pragmatics v2 artifact visibly repairs a defective pair and changes the reported strict-pair denominator, demonstrating useful evidence correction rather than silent smoothing.
- The coding-benchmark and PHREEQC papers explicitly show that tool or model improvements can exchange which items are correct, a practical warning for release decisions.
- The quantum paper is short but visually clear: architecture, circuit layers, and result curves align with the described mechanism.

## Weaknesses

- No experiment was independently rerun, and several sources expose incomplete reproduction surfaces.
- AutoMem uses only three games, per-environment optimization, a small fixed seed list, and proprietary meta-LLMs.
- Self-GC relies on private traces and a judge-based counterfactual metric; its online split is not a randomized quality experiment.
- SPS uses one pretraining corpus, one main seed, sub-2B models, and approximately doubles per-step training compute.
- OpenAgent is a controlled sandbox; the reported perturbation intervention does not consistently eliminate degradation at later training stages.
- Adversarial pragmatics uses 18 items, three local models, one author-adjudicator, toy payloads, and a judge that scores its own outputs.
- SkillDepAnalyzer is both measurement instrument and research artifact, and its public-snapshot corpus excludes private and enterprise skills.
- The coding-benchmark public repository omits raw replays and the internal pipeline, limiting exact reproduction.
- RareDxR1 lacks inspectable code, data, weights, preprocessing, clinical governance, and prospective validation at the public pin.
- PHREEQC-MCQ-200 uses multiple-choice tasks from 21 scenario stems, and the inspected sources do not expose an official public artifact locator.
- The quantum paper emulates links rather than operating physical fiber-interconnected QPUs and supplies no uncertainty bars, raw counts, or calibration snapshot.
- Cross-domain synthesis risks overgeneralization; similar reliability vocabulary does not make all domains interchangeable.

## Potential Implementations

### Reliability manifest compiler

- `User`: Agent-platform and evaluation engineers.
- `Goal`: Produce one signed, reviewable manifest for an agent run without collapsing different evidence types.
- `Core mechanism`: Collect version pins, typed context objects, skill/package/service dependencies, tool schemas, judge rubrics, per-item outcomes, and final artifact hashes.
- `Required inputs`: Repository commits, model identifiers, dependency graph, tool contract, evaluation records, and public-safe source locators.
- `Outputs`: JSON manifest, Markdown evidence summary, missing-evidence report, and sanitization results.
- `Risk controls`: Secret scanning, allowlisted fields, path sanitization, no raw private traces, signed correction lineage.
- `Evaluation`: Rebuild one report table and verify that a second reviewer can trace each claim to an immutable source.

### Context retention testbench

- `User`: Developers of long-horizon agents.
- `Goal`: Compare learned memory, object-level garbage collection, and heuristic pruning on the same tasks.
- `Core mechanism`: Synthetic repository tasks contain hidden future dependencies, editable artifacts, delayed constraints, and exact recovery checks.
- `Required inputs`: Public synthetic traces, state-policy adapters, fixed seeds, and deterministic invariant tests.
- `Outputs`: Compression, recovery, retained-item, error-transition, privacy, and cost vectors.
- `Risk controls`: No user histories, no external side effects, no credential-shaped data, fail-closed latest-instruction retention.
- `Evaluation`: Pre-register tasks and require exact locator/artifact recovery plus independent judge review for semantic continuations.

### Tool-shift and evaluator audit harness

- `User`: Safety and scientific-agent evaluators.
- `Goal`: Separate environment-shift failure, tool-protocol failure, unsafe instruction uptake, and judge failure.
- `Core mechanism`: Combine benign OpenAgent-style shifts, PHREEQC-style output interfaces, and adversarial-pragmatics labels over harmless fixtures.
- `Required inputs`: Synthetic tools, frozen deterministic simulator, paired instruction/data cases, independent rubrics.
- `Outputs`: Per-tier accuracy, item retention, failure stage, policy compliance, judge minority recall, and abstention rate.
- `Risk controls`: Toy payloads, deterministic tools, no harmful content, read-only workspaces, independent adjudication.
- `Evaluation`: A release passes only if aggregate gains do not exceed a pre-set lost-item budget and judge minority recall clears a pre-registered threshold.

### Skill dependency gate

- `User`: Skill registries, package managers, and enterprise agent administrators.
- `Goal`: Prevent installation of ambiguous or high-risk transitive skill dependencies.
- `Core mechanism`: Resolve skill, package, service, permission, and data-store edges; compare pins against policy; block missing identity or license evidence.
- `Required inputs`: Skill bundle, AgentBOM schema, registry metadata, package lockfiles, service declarations, and policy.
- `Outputs`: Dependency graph, risk warnings, unresolved-edge queue, and signed lock record.
- `Risk controls`: Static analysis by default, no execution of untrusted installers, sandboxed optional verification, explicit approval for network or shell authority.
- `Evaluation`: Hand-labeled graphs, known safe fixtures, known ambiguous fixtures, and false-positive/false-negative review.

### High-stakes evidence gate

- `User`: Clinical AI and quantum-system research teams.
- `Goal`: Prevent benchmark or emulation results from being presented as deployment readiness.
- `Core mechanism`: Require domain-specific cards before promotion: clinical data lineage, calibration, subgroup and prospective review; or quantum calibration, raw counts, uncertainty, post-selection yield, and physical-link evidence.
- `Required inputs`: Study protocol, domain metadata, raw-to-summary lineage, independent review, and deployment claim.
- `Outputs`: Claim classification, satisfied/unsatisfied evidence gates, and permitted public wording.
- `Risk controls`: No automated clinical decisions, no patient data in public outputs, no hardware control, conservative claim templates.
- `Evaluation`: Domain experts review whether each promoted claim is entailed by the submitted evidence.

## Three Ways to Exercise This Research

1. `Build a synthetic state-retention matrix`: Objective—compare sliding-window, learned file memory, and object-level lifecycle control. Inputs—a public toy repository, delayed requirements, exact file locators, and fixed seeds. Method—run each policy in a no-egress sandbox, record tokens removed, future dependencies retained, artifacts recovered, and old-correct/new-wrong transitions. Output—a vector report rather than a single score. Success criterion—every reported compression result includes exact recovery and item-retention measures. Stop condition—halt on any secret-shaped fixture, non-deterministic side effect, or missing run manifest.
2. `Audit one agent skill without executing it`: Objective—construct a typed dependency and authority graph. Inputs—one public skill bundle, its referenced skills, package manifests, service URLs, and license files. Method—parse metadata, prose, and scripts statically; resolve skill/package/service edges; pin public sources; classify unresolved identities and requested permissions. Output—a public-safe AgentBOM and reviewer queue. Success criterion—a second reviewer can reproduce every edge from cited evidence. Stop condition—do not run installers, scripts, remote services, or ambiguous dependencies.
3. `Re-score a deterministic tool benchmark`: Objective—show how aggregate accuracy changes under retained/gained/lost accounting and alternative penalty rules. Inputs—a small public deterministic simulator fixture, fixed answer key, two output-access protocols, and synthetic model traces. Method—compute accuracy, retention, failure stage, token cost, and rank under at least two transparent scoring rules. Output—a score-sensitivity report with per-item traces. Success criterion—every rank change is attributable to task outcomes or an explicit scoring choice. Stop condition—halt if the simulator version, data hash, or output parser is not pinned.

## Example MVP Product

- `Product name`: Reliability Ledger
- `Target user`: Teams releasing agent workflows, evaluation suites, or research-backed technical products.
- `Problem`: Run state, tool behavior, dependency provenance, evaluator judgments, and source evidence are stored in different formats, so a strong aggregate result can be published without exposing lost items or missing reproducibility.
- `Core workflow`: Import a run manifest and public source list; resolve version pins and typed dependencies; ingest per-item outcomes; classify failure stages and judge labels; scan for private path or identity leaks; generate a signed Markdown/JSON review packet; require approval before a claim is promoted.
- `Data requirements`: Public repository metadata, explicitly authorized run summaries, hashed artifact inventories, tool contracts, dependency manifests, rubric definitions, and per-item outcomes. Raw private traces and clinical records are excluded from the MVP.
- `Architecture`: Local-first CLI plus static web report; schema validators; Git and URL resolver; AgentBOM graph builder; per-item retention calculator; judge-confusion analyzer; sanitization gate; signature and correction ledger.
- `Success metrics`: 100% source-pin coverage for promoted claims; 100% major claims mapped to evidence; zero sanitization leaks; per-item retention reported for every comparative release; independent reviewer trace success above 95%; no scalar score without visible component metrics.
- `Risk controls`: Read-only source access; no execution of untrusted skills; no external writes without approval; allowlisted public fields; secret and local-path scanning; deletion policy for temporary files; explicit domain gate for medical or hardware claims.
- `Limitations`: The MVP verifies evidence structure, not scientific truth; static dependency extraction can miss dynamic imports; URLs can disappear; independent domain review remains necessary.
- `MVP boundary`: No clinical decision support, model training, automatic benchmark execution, vulnerability exploitation, or quantum-hardware control.
- `Deployment model`: Local CLI generating a reviewable static artifact and optional CI status.
- `Evaluation plan`: Synthetic fixtures for missing pins, lost-item regressions, judge minority failures, ambiguous dependencies, sanitization leaks, and correction lineage; then a blinded reviewer study on public runs.
- `Failure modes`: False confidence from complete metadata around weak evidence, stale repository pins, incomplete dynamic dependencies, schema gaming, and over-aggregation.
- `Maintenance plan`: Version the schema, refresh resolver rules, preserve correction history, and require periodic domain-policy review.
